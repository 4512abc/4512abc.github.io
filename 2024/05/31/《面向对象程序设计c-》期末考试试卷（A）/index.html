《面向对象程序设计c++》期末考试试卷（A）

一、选择题（每小题2分，共40分）

1、c++是（ ）。

A. 面向对象的程序设计语言 B. 面向过程的程序设计语言

c. 既支持面向对象的程序设计又支持面向过程的程序设计的混合型语言 D. 非结构化的程序设计语言

2、面向对象程序设计思想的主要特征中不包括（ ）。

A. 封装性 B. 多态性 c. 继承性 D. 功能分解，逐步求精

  1. C++是（C）既支持面向对象的程序设计又支持面向过程的程序设计的混合型语言。C++是一种多范式编程语言，既支持面向对象编程（OOP）的特性，如封装、继承、多态，也支持面向过程编程的特性，如函数、过程、结构体等。这使得C++具有很高的灵活性和通用性，可以用于多种编程范式。

  2. （D）功能分解，逐步求精不是面向对象程序设计思想的主要特征。面向对象程序设计思想的主要特征包括封装性（Encapsulation）、多态性（Polymorphism）、继承性（Inheritance）。功能分解，逐步求精是一种程序设计方法，它强调将大的问题分解成小的、可管理的部分，并逐步完善每个部分，但这不是面向对象程序设计的特征。

3、若定义：string str; 当语句cin>>str; 执行时，从键盘输入： Microsoft Visual Studio 6.0! 所得的结果是str=（ ）。

A. Microsoft Visual Studio 6.0! B. Microsoft

c. Microsoft Visual D. Microsoft Visual Studio 6.0

4、考虑下面的函数原型声明：void testDefaulParam(int a,int b=7,char z=”*”); 下面函数调用中，不合法的是（ ）。

A. testDefaulParam(5); B. testDefaulParam(5,8); c. testDefaulParam(5,”#”); D. testDefaulParam(0,0,”*”);

  1. 答案是（B）Microsoft。C++中的输入操作符>>会在遇到空白字符（空格、制表符、换行符等）时停止读取输入，因此只会读取到第一个空白字符之前的内容。在这种情况下，cin >> str;只会读取输入的第一个单词”Microsoft”，并将其存储到字符串str中。所以，str的值是”Microsoft”。

  2. 不合法的函数调用是（C）testDefaulParam(5,"#")。在C++中，默认参数只能在函数声明（包括函数原型声明）中指定，而不能在函数定义中重新指定。在函数声明void testDefaulParam(int a, int b=7, char z="*");中，参数b和z已经有了默认值，分别是7和”*”。因此，只有在调用函数时省略了有默认值的参数时，才可以。而在C选项中，试图在调用时为z参数重新指定默认值，这是不合法的。

5、下列语句中，将函数int sum(int x, int y)正确重载的是（ ）。

A. float sum(int x, int y); B. int sum(int a, int b); c. float sum(float x, float y); D. double sum(int y, int x);

6、下列表示引用的方法中，（ ）是正确的。 已知：int a=1000;

A. int &x=a; B. char &y; c. int &z=1000; D. float &t=&a;

  1. 正确重载函数的选项是（C）int sum(int a, int b);。函数的重载是指在同一个作用域内，函数名相同但参数列表不同的多个函数

  2. 正确的引用方法是（A）int &x=a;。引用必须在声明时初始化，并且引用的类型必须与其引用的变量类型相同。所以，选项A中，int &x=a;声明了一个整型引用x，并将其初始化为整型变量a的引用，这是正确的用法。

选项B中，char &y;缺少初始化，引用必须在声明时初始化。

选项C中，int &z=1000;试图将一个整数字面值初始化为一个整型引用，这是不合法的，因为引用必须引用一个实际的对象。

选项D中，float &t=&a;试图将一个指向整型变量的指针初始化为一个浮点型引用，这也是不合法的。

7、在一个函数中，要求通过函数来实现一种不太复杂的功能，并且要求加快执行速度，选用（ ）。

A. 内联函数 B. 重载函数 c. 递归调用 D. 嵌套调用

8、下列有关c++类的说法中，不正确的是（ ）。

A. 类是一种用户自定义的数据类型

B. 只有类中的成员函数或类的友元函数才能存取类中的私有成员

c. 在类中，如果不做特别说明，所有成员的访问权限均为私有的

D. 在类中，如果不做特别说明，所有成员的访问权限均为公用的

  1. 选项（A）内联函数。内联函数适合在要求执行速度较快且功能不太复杂的情况下使用。内联函数会在编译时将函数的调用处直接替换为函数体的代码，从而减少了函数调用的开销，提高了执行速度。

  2. 不正确的是选项（D）在类中，如果不做特别说明，所有成员的访问权限均为私有的。

9、已知X类，则当程序执行到语句：X array[3];时，调用了（ ）次构造函数。

A. 0 B. 1 c. 2 D. 3

10、下面说法中，正确的是（ ）

A. 一个类只能定义一个构造函数，但可以定义多个析构函数

B. 一个类只能定义一个析构函数，但可以定义多个构造函数

c. 构造函数与析构函数同名，只要名字前加了一个求反符号（~）

D. 构造函数可以指定返回类型，而析构函数不能指定任何返回类型，即使是void

类型也不可以

  1. 当程序执行到语句X array[3];时，调用了（D）3 次构造函数。这是因为这条语句声明了一个名为array的数组，数组的元素类型是X类，而数组长度为 3。在C++中，当创建数组时，每个数组元素都会调用一次默认构造函数来进行初始化，因此在这里会调用 3 次X类的构造函数。

  2. 正确的说法是（B）一个类只能定义一个析构函数，但可以定义多个构造函数。在C++中，一个类可以有多个构造函数，这就是构造函数的重载。通过构造函数的重载，可以实现不同的初始化方式。而析构函数只能有一个，用来释放对象所占用的资源。选项C中的说法不正确，析构函数与构造函数的命名规则不同，析构函数的命名是在类名前面加一个波浪号~，并且不能加反斜杠。选项D中的说法也不正确，构造函数不能指定返回类型，包括void类型。

11、已知：print( )函数是一个类的常成员函数，它无返回值，下列表示中，（ ）是正确的。

A. void print( ) const; B. const void print( ); c. void const print( ); D. void print(const);

12、下面描述中，表达错误的是（ ）

A. 公用继承时基类中的public成员在派生类中仍是public的

B. 公用继承时基类中的private成员在派生类中仍是private的

c. 公用继承时基类中的protected成员在派生类中仍是protected的

D. 私有继承时基类中的public成员在派生类中是private的

  1. 正确的是选项（A）void print() const;。这是一个声明了const限定符的成员函数，表示该成员函数是一个常成员函数，它不会修改对象的状态。在声明常成员函数时，const关键字应该紧跟在函数的参数列表后面。

  2. B是错误的。

    在公有继承中，派生类会继承基类的公有和受保护成员，但不会继承基类的私有成员。

    所以选项 B 表达的是错误的，私有继承时基类中的私有成员在派生类中是不可见的。

知识点：基类：public; protect; private

派生类公有继承基类：public; protect; 不可访问;

派生类私有继承基类：private; private; 不可访问；

公有继承的情况下，基类的public，protect成员的属性在派生类中保持不变，private成员的属性变为不可访问即派生类只能通过基类的成员函数访问基类的私有成员。

私有继承的情况下，基类的public，protect成员的属性在派生类中变成私有，private成员的属性变为不可访问即派生类只能通过基类的成员函数访问基类的私有成员。

13、设置虚基类的目的是（ ）。

A. 简化程序 B. 消除二义性 c. 提高运行效率 D. 减少目标代码

14、下面（ ）的叙述不符合赋值兼容规则。

A. 派生类的对象可以赋值给基类的对象

B. 基类的对象可以赋值给派生类的对象

c. 派生类的对象可以初始化基类的对象

D. 派生类的对象的地址可以赋值给指向基类的指针

  1. 设置虚基类的目的是（B）消除二义性。在多重继承中，如果某个类同时继承了多个共同的基类，而这些基类又有一个共同的基类，就可能出现二义性问题。为了消除这种二义性，可以将这个共同的基类设置为虚基类，这样在派生类中只会包含一份虚基类的实例，从而避免了二义性。

  2. B是错误的。

    在C++中，基类的对象不能直接赋值给派生类的对象。这是因为派生类的对象可能包含基类没有的成员或者行为，因此基类的对象无法完全赋值给派生类的对象。

    所以选项 B 表达的是错误的。

15、关于虚函数的描述中，（ ）是正确的。

A. 虚函数是一个static类型的成员函数 B. 虚函数是一个非成员函数

c. 基类中说明了虚函数后，派生类中与其对应的函数可不必说明为虚函数
D. 派生类的虚函数与基类的虚函数具有不同的参数个数和类型

16、下面关于友元的描述中，错误的是（ ）。

A. 友元函数可以访问该类的私有数据成员

B. 一个类的友元类中的成员函数都是这个类的友元函数

c. 友元可以提高程序的运行效率

D. 类与类之间的友元关系可以继承

15、关于虚函数的描述中，（C）是正确的。基类中声明的虚函数，在派生类中可以不必重新声明为虚函数。如果基类中的函数被声明为虚函数，它在派生类中的重写版本也会自动成为虚函数，无需再次使用关键字 virtual 进行声明。

16、下面关于友元的描述中，错误的是（D）

关于选项D，类与类之间的友元关系不能被继承。友元关系是一种单向的关系，不能被继承。如果类A是类B的友元，那么类B并不自动成为类A的友元，反之亦然。

17、下列关于静态数据成员的说法，不正确的是（ ）。

A. 类中定义的公用静态数据成员，可以通过类的对象来访问

B. 类中定义的所有静态数据成员，都必须在类外初始化

c. 静态数据成员不是所有对象所共用的

D. 普通的成员函数可以直接访问类中的静态数据成员

答案是C，是共用的。

18、如果表达式++i*k中的“++”和“*”都是重载的友元运算符，若采用运算符函数调用格式，则表达式还可以表示为（ ）。

A. operator*(i.operator++(),k)

B. operator*(operator++(i),k)

C. i.operator++().operator*(k)

D. k.operator*(operator++(i))

对于表达式 ++i * k，如果 ++ 和 * 都是重载的友元运算符，则根据重载运算符函数的调用格式，它们会被转换为相应的函数调用。

对于前置递增运算符 ++i，重载后的函数调用格式应该是 operator++(i)。

对于乘法运算符 *，重载后的函数调用格式应该是 operator*(lhs, rhs)，其中 lhs 是左操作数，rhs 是右操作数。

因此，表达式可以重写为 operator++(i) * k，这与选项 B 相匹配。

20、下列的各类函数中，（ ）不是类的成员函数

A. 构造函数 B. 析构函数 c. 友元函数 D. 复制构造函数

正确答案是 C. 友元函数。

构造函数（A）、析构函数（B）和复制构造函数（D）都是类的成员函数，它们分别用于对象的构造、析构和复制。它们都属于类的一部分，因此是类的成员函数。

而友元函数（C）虽然可以访问类的私有成员，但它并不属于类的成员函数。友元函数是在类外部声明的函数，可以被授权访问类的私有成员。
