计算机组成原理之存储系统

存储系统概述

1. 存储器的层次结构

存储器的层次结构旨在平衡速度、容量和成本。常见的层次结构包括：

  1. Cache（高速缓存）：

    • 功能：解决主存与CPU之间速度不匹配的问题。
    • 特点：速度快，容量小，成本高。
    • 位置：位于CPU与主存之间。

  2. 主存（RAM）：

    • 功能：存储正在执行的程序和数据。
    • 特点：速度较快，容量较大，但相对于磁盘等外存，仍有限。
    • 位置：位于CPU和外存之间。

  3. 辅存（外存）：

    • 功能：存储大量数据和程序，解决主存容量不足的问题。
    • 特点：速度较慢，但容量大且成本较低。
    • 类型：包括硬盘、固态硬盘、光盘等。

2. 存储器的分类

存储器可以根据不同的标准进行分类：

  1. 按层次进行分类：

    • 缓存（Cache）：提供最快的存取速度，主要用于存放频繁访问的数据。
    • 主存（RAM）：提供较大的存储容量和较快的存取速度。
    • 外存（硬盘、光盘等）：提供大容量的存储，但速度较慢。

  2. 按传输介质进行分类：

    • 半导体存储器：如主存（RAM）、缓存（Cache）。特点是速度快，成本高。
    • 磁表面存储器：如磁盘（硬盘）、磁带。特点是容量大，速度较慢。
    • 光存储器：如光盘。特点是容量适中，读写速度慢。

  3. 按存取方式进行分类：

    • 相联存储器（Associative Memory）：可以根据内容进行检索，如内容地址存储器（CAM）。
    • 随机存取存储器（RAM, Random Access Memory）：读写任何一个存储单元的时间相同，与位置无关。
    • 顺序存取存储器（SAM, Sequential Access Memory）：读写时间与位置相关，如磁带。
    • 直接存取存储器（DAM, Direct Access Memory）：结合了随机存取和顺序存取特性，如机械硬盘。

  4. 按信息的可更改性进行分类：

    • 读写存储器（Read/Write Memory）：可读可写，如内存、硬盘。
    • 只读存储器（ROM, Read-Only Memory）：只能读取数据，如光盘。

  5. 按信息的可保存性进行分类：

    • 易失性存储器（Volatile Memory）：断电后数据丢失，如主存（RAM）、Cache。
    • 非易失性存储器（Non-Volatile Memory）：断电后数据仍然保存，如硬盘、光盘。
    • 破坏性读出（Destructive Readout）：读出数据后需要重写，如DRAM。
    • 非破坏性读出（Non-Destructive Readout）：读出数据后不破坏原数据，如SRAM、磁盘、光盘。

3. 存储器的性能指标

  1. 存储容量：

    • 定义：存储器可以存储的总数据量，通常以字节（Byte）、千字节（KB）、兆字节（MB）、千兆字节（GB）等为单位。
    • 计算：存储容量 = 存储字数 × 存储字长。比如，1M × 8 位 表示 1 兆字节的存储容量，每个存储单元为 8 位。

  2. 单位成本：

    • 定义：每单位存储的成本，通常以每位或每字节的价格来衡量。
    • 计算：单位成本 = 总成本 / 总容量。例如，某条内存条价格为 259 元，容量为 8GB，则单位成本 = 259 / (8 × 1024) 元/GB。

  3. 存储速度：

    • 数据传输率（Data Transfer Rate）：指单位时间内可以传输的数据量，通常以字/秒、字节/秒、位/秒等为单位。
    • 计算：数据传输率 = 数据宽度 / 存储周期。

  4. 存取时间：

    • 定义：从发起存储操作到完成操作的时间。
    • 计算：存取时间 = 存取周期 - 恢复时间。存取周期是进行一次读写操作所需的时间，包括存取时间和恢复时间。

  5. 主存带宽：

    • 定义：每秒钟从主存中读写信息的最大数量。
    • 计算：主存带宽 = 数据传输率 × 存取周期。主存带宽表示主存的处理能力，通常以字/秒、字节/秒、位/秒为单位。

主存储器

主存储器的基本组成

主存储器（RAM）的基本组成部分包括存储体、地址寄存器和数据寄存器，以及控制电路。

  1. 存储体

    • 存储单元（Memory Cell）: 存储体由多个存储单元组成，每个存储单元可以存储一个比特。存储单元由存储元构成，存储元可以是电容（用于DRAM）或双稳态触发器（用于SRAM）。
    • 电容: 用于DRAM存储信息。电容内有电荷表示比特1，无电荷表示比特0。电荷的保持时间较短（通常为2ms），需要定期刷新以保持数据。
    • MOS管: 用作开关，控制电容的充放电，MOS管的开关状态决定电容的电荷是否被释放。

  2. 地址寄存器和数据寄存器

    • 地址寄存器（MAR, Memory Address Register）: 存储要访问的内存地址。
    • 数据寄存器（MDR, Memory Data Register）: 存储从存储体中读取或将要写入存储体的数据。

  3. 控制电路

    • 控制电路: 管理地址寄存器、数据寄存器的操作，控制数据的读写过程。
    • 片选线、读选择线、写选择线: 用于选择具体的存储芯片，控制读写操作。片选线决定是否激活该芯片，读选择线和写选择线决定数据操作的类型（读或写）。

  4. 寻址方式

    • 按字节寻址: 每个存储单元按字节进行寻址。
    • 按字寻址: 存储单位为“字”，字长为固定长度（如4字节）。
    • 按半字寻址: 每个存储单元按半字（如2字节）进行寻址。
    • 按双字寻址: 每个存储单元按双字（如8字节）进行寻址。

随机存取存储器（RAM）

RAM主要有两种类型：SRAM（静态RAM）和DRAM（动态RAM）。

  1. SRAM（静态RAM）

    • 存储元件: 使用双稳态触发器（flip-flop）来存储比特。
    • 特点: 非破坏性读出，读写速度快，适用于Cache。成本较高，功耗较大。
    • 双稳态触发器: 通过两个稳定的状态表示1和0，读出时不会破坏原有数据。

  2. DRAM（动态RAM）

    • 存储元件: 使用电容和MOS管来存储信息。
    • 特点: 需要定期刷新，读出数据时电容中的电荷会放电（破坏性读出），刷新时间通常为2ms。制造成本低，集成度高，功耗低。
    • 刷新机制: 每隔2ms刷新一次，每次刷新一行存储单元。刷新可以通过硬件自动进行。

  3. 地址线复用技术

    • 行地址和列地址复用: 为了减少地址线的数量，行地址和列地址分两次传送到译码器中。这样可以显著减少所需的地址线数量。

只读存储器（ROM）

ROM是一种只读存储器，但有些ROM（如闪存）也支持写操作。ROM通常用于存储固件（如BIOS），这些内容通常不需要频繁更改。

  1. 闪存: 读写速度较慢，但支持可擦除和可编程。写入操作前需要擦除。
  2. ROM特性: 传统ROM为只读，但现代ROM（如EEPROM）允许一定程度的写入操作。

主存储器和CPU的连接

  1. 单块存储芯片与CPU的连接

    • 存储芯片规格: 例如，8Kx1位的存储芯片需要13根地址线。CPU有8个数据线和16根地址线。单块芯片无法充分利用所有数据线，通常需要多个存储芯片来扩展主存容量。
    • 扩展方式:
      • 位扩展: 扩展存储单元的字长（如将1位扩展为8位）。
      • 字扩展: 扩展存储单元的数量（如增加存储单元的数量）。

  2. 多块存储芯片与CPU的连接

    • 位扩展: 通过将多个存储芯片连接在一起，实现更大的存储宽度。例如，将两个8Kx1位芯片连接成8Kx2位的存储器。
    • 字扩展: 使用地址线选择不同的存储芯片。例如，通过地址线A13和A14来选择不同的存储芯片。

  3. 双端口RAM和多模块存储器

    • 双端口RAM: 具有两个独立的数据线、地址线和控制线，允许同时进行两次读写操作。适用于多核CPU的环境。
    • 多模块存储器:
      • 多体并行存储器: 通过低位交叉编址实现并行读取，提高存储带宽。
      • 单体多字存储器: 将多个存储体合并，每次同时取出多个字，适用于大容量存储需求。

  4. 应用

    • 双通道内存: 采用低位交叉编址的多体并行存储器，以提高内存带宽和性能。

Cache

Cache的基本概念和原理

知识总览：

尽管双端口RAM和多模块存储器已经提高了存储器的工作速度，但与CPU的速度差距仍然很大，因此产生了缓存(Cache)。Cache的价格较高、容量较低，但能显著提高存取速度。

（1）工作原理：

  • 空间局部性：在最近的未来，CPU可能会访问到与当前访问数据存储在相邻位置的信息。这是因为数据通常按顺序存储。

  • 时间局部性：在最近的未来，CPU可能会再次访问当前正在使用的数据。这是由于程序中存在大量循环和重复访问的数据。

基于这两个局部性原理，我们可以将CPU当前访问的地址周围的数据预先放入Cache中，从而提高访问速度。

（2）性能分析：

  • 设 ($ t_c$ ) 为访问一次Cache所需时间，( $t_m$ ) 为访问一次主存所需时间。

  • 命中：当CPU可以在Cache中直接找到所需的信息。

  • 命中率 ( H )：CPU请求的信息在Cache中的比例。

  • 缺失率（未命中率）( M )：( M = 1 - H )

  • CPU平均访问时间 ( t )：($ t = H \cdot t_c + (1 - H) \cdot (t_c + t_m$) )

例题：

在计算Cache性能时，需要根据命中率和访问时间来评估实际效果。

（3）如何界定“周围”：

主存的存储空间被分成块（Block），Cache也以块为单位进行数据交换。这使得主存地址可以分为块号和块内地址。操作系统中，一个块也称为页面或页框，Cache中的块也称为行。

总结：

Cache系统通过存储最近使用的数据来提高速度，利用空间局部性和时间局部性来减少主存访问次数。

Cache和主存的映射方式

（1）全相联映射

  • 定义：主存中的任何一个块可以被存放在Cache的任意位置。

  • 实现：每个Cache块有一个标记位，用于记录主存块号。还需要一个有效位，用于标记Cache中的数据是否有效。

  • 优点：Cache的存储空间利用充分，命中率高。

  • 缺点：查找速度较慢，需要对比所有Cache块的标记位。

（2）直接映射

  • 定义：主存中的每个块只能映射到Cache中的一个特定位置。计算方法为：Cache块号 = 主存块号 % Cache总块数。

  • 实现：如果Cache总块数为 ( 2^n )，主存块号的末尾 ( n ) 位直接反映它在Cache中的位置。

  • 优点：只需要对比一个标记位，速度最快。

  • 缺点：Cache存储空间利用不充分，命中率较低。

（3）组相联映射

  • 定义：Cache块被分成若干组，每个主存块可以放在特定分组中的任意位置。主存块对应的组号 = 主存块号 % 分组数。

  • 实现：如果Cache总分组数为 ( 2^n )，主存块号的末尾 ( n ) 位决定其在Cache中的组号。

  • 优点：综合了全相联映射和直接映射的优点，提供了更好的性能和存储空间利用率。

总结：

不同的映射方式在存储空间利用率和查找速度上各有优劣，选择合适的映射方式可以提高Cache的性能。

Cache替换算法

知识总览：

当Cache满时，需要选择一个块进行替换。以下是几种常见的替换算法：

  1. 随机算法（RAND）

    • 定义：随机选择一个块进行替换。

    • 优点：实现简单。

    • 缺点：没有考虑局部性原理，命中率低，效果不稳定。

  2. 先进先出算法（FIFO）

    • 定义：替换最先进入Cache的块。

    • 优点：实现简单。

    • 缺点：可能会替换掉频繁访问的数据，导致“抖动现象”。

  3. 近期最少使用（LRU）

    • 定义：记录每个块的使用时间，替换最久未被访问的块。

    • 优点：基于局部性原理，通常能提供较高的命中率。

    • 缺点：实现复杂，需要额外的计数器或记录机制。

  4. 最近不经常使用（LFU）

    • 定义：记录每个块被访问的次数，替换访问次数最少的块。

    • 优点：适用于访问模式稳定的场景。

    • 缺点：可能无法很好地处理频繁访问的数据，效果不如LRU。

总结：

不同的替换算法在性能和实现复杂性上各有不同，需要根据具体情况选择合适的算法。

Cache写策略

知识总览：

Cache的写策略涉及如何处理写入操作，以保证数据的一致性和优化性能。

  1. 写命中

    • 写回法：只修改Cache中的数据，只有在Cache块被替换时才写回主存。需要使用脏位标记数据是否被修改。

    • 全写法：在Cache和主存中都进行写入操作，通常使用写缓冲来优化性能。

    • 写缓冲：写缓冲提高了写操作的速度，但如果写操作频繁，可能导致缓冲区饱和。

  2. 写不命中

    • 写分配法：当Cache写不命中时，将主存中的块调入Cache，并在Cache中修改，通常搭配写回法使用。

    • 非写分配法：直接写入主存，不将主存块调入Cache，通常搭配全写法使用。

  3. 多级Cache

    • 定义：现代计算机使用多级Cache，其中离CPU越近的Cache速度越快、容量越小，离CPU越远的Cache速度越慢、容量越大。

    • 策略：各级Cache之间通常采用全写法 + 非写分配法，而Cache和主存之间通常采用写回法 + 写分配法。

虚拟存储系统

虚拟存储系统（Virtual Memory System）是现代计算机系统中关键的一部分，它使得程序可以使用比实际物理内存更大的地址空间，并提供了对内存的有效管理。它主要通过页式存储、段式存储和段页式存储来实现。

页式存储

页式存储是一种将程序和数据分成固定大小的块（页面）的内存管理技术，使得程序能够非连续地存储在物理内存中。

基本概念：

  • 逻辑地址（虚拟地址）：由程序员看到的地址。
  • 物理地址（实地址）：实际在主存中的地址。
  • 页表：记录每个逻辑页面（虚拟页面）在主存中的位置。页表存储在主存中，用于将逻辑地址转换为物理地址。

地址变换过程：

  1. 拆分逻辑地址：将逻辑地址拆分为逻辑页号和页内地址。
  2. 查找页表：通过逻辑页号查找页表，得到对应的物理主存块号。
  3. 组合物理地址：将物理主存块号与页内地址组合得到最终的物理地址。
  4. Cache查询：到Cache中查找物理地址，如果没有则访问主存。

快表（TLB, Translation Lookaside Buffer）：

  • 定义：快表是一种特殊的高速缓存，用于存储最近访问的页表项，以加快地址转换速度。
  • 区别：
    • 快表：存储页表项的副本，使用SRAM，访问速度更快。
    • Cache：存储主存块的副本。

页式虚拟存储器：

  • 辅存（例如硬盘）中的数据分为块，类似于Cache中的块。
  • 逻辑页号：逻辑地址中的页号。
  • 主存块号：物理地址中的主存块号。
  • 外存块号：辅存中的块号。
  • 有效位：指示页面是否已调入主存。
  • 访问位：用于页面替换算法。
  • 脏位：指示页面是否被修改过，需要更新主存和辅存之间的数据一致性。

虚拟存储器的优势：

  • 解决了主存容量不足的问题，使得程序可以使用比物理内存更大的虚拟地址空间。
  • Cache—主存：解决了主存与CPU之间速度不匹配的问题。

段式虚拟存储器

段式虚拟存储器将程序划分为功能模块（段），每个段可以独立地存储在内存中。与页式存储相比，段式存储更关注程序的逻辑结构而不是物理结构。

基本概念：

  • 段：程序的逻辑模块，例如代码段、数据段、栈段等。
  • 段表：记录每个段的起始地址和长度，操作系统用于决定哪些段应该被调入内存。

段式虚拟存储器的优点：

  • 适合程序逻辑结构清晰的情况。
  • 允许按需加载和替换段，提高了内存利用率。

段页式虚拟存储器

段页式虚拟存储器结合了段式存储和页式存储的优点，将程序先按功能模块分段，每段再划分为固定大小的页。主存空间也划分为大小相等的页。

基本概念：

  • 段表：每个程序有一个段表，记录段的起始地址和长度。
  • 页表：每段有一个页表，记录页在主存中的位置。

地址变换过程：

  1. 拆分虚拟地址：

    • 段号：虚拟地址中的段号。
    • 段内页号：段内的页号。
    • 页内地址：页内的具体地址。

  2. 查找段表：通过段号查找段表，得到段的起始地址。

  3. 查找页表：通过段内页号查找页表，得到页在主存中的位置。
  4. 组合物理地址：将页在主存中的位置与页内地址组合，得到最终的物理地址。

优势：

  • 结合了段式存储和页式存储的优点，提供了灵活的内存管理方式。
  • 允许按需加载和替换，更加高效地使用内存。
