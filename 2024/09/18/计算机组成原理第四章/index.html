计算机组成原理之指令系统

一：指令系统的发展与性能要求

1. 指令系统及其发展

指令系统（或称指令集）是指一台计算机中所有机器指令的集合。它是计算机硬件与软件之间的接口，定义了计算机能执行的操作及其格式。指令系统的设计直接影响计算机的硬件结构、系统软件的开发和计算机的性能。

指令系统的基本组成包括：

  • 操作码（Opcode）：指令中指定操作的部分。
  • 操作数（Operand）：指令中指定操作对象的部分。
  • 寻址方式：指定操作数的位置或存取方式。

指令系统的发展历经了多个阶段，从最初的简单指令集到复杂的现代指令集，不断演进以满足更高的性能和灵活性要求。现代计算机指令系统的发展趋势包括：

  • 从简单到复杂：早期计算机指令集较为简单，后期发展出复杂指令集以支持更多功能和提高效率。
  • 从专用到通用：早期的指令集往往针对特定应用设计，而现代指令集更加通用，支持多种应用和操作。
  • 支持多核和并行处理：现代指令集支持多核处理和并行计算，以提高处理性能。

2. 指令系统性能的要求

一个完善的指令系统应满足以下四方面的要求：

  1. 完备性：指令系统应提供足够的指令来编写各种程序，无需依赖软件模拟指令。这意味着指令集应该足够全面，能够覆盖计算机系统的各种操作需求。

  2. 有效性：指令系统应能使程序在计算机上高效运行，主要体现在以下几个方面：

    • 程序占据存储空间小：指令系统应设计合理，避免冗余，减少程序代码的长度。
    • 执行速度快：指令系统的设计应确保指令能够快速执行，以提高整体系统性能。

  3. 规整性：规整性包括：

    • 对称性：指令系统中各指令应具有相似的格式和操作方式，降低编程复杂度。
    • 匀齐性：指令系统中的指令应均匀分布，避免某些功能过于复杂而其他功能过于简单。
    • 指令格式和数据格式的一致性：指令格式和数据格式应保持一致，简化程序设计和实现。

  4. 兼容性：指令系统应具备向下兼容性，即系列机各机种之间应具有相同的基本结构和共同的基本指令集，以确保低档机上运行的软件可以在高档机上运行。这有助于保护软件投资，并简化软件的移植工作。

3. 常见的指令集

以下是几种常见的指令集的简要介绍：

  1. x86 指令集（复杂指令集）：

    • 特点：x86 指令集是一个复杂指令集架构（CISC），它包含了大量的指令，支持复杂的操作和多种寻址方式。它在个人电脑和服务器中广泛使用。
    • 历史背景：由Intel公司开发，最初的x86指令集出现于1978年，并随着时间的推移不断扩展和增强。
    • 现代化：现代的x86处理器（如Intel的Core系列和AMD的Ryzen系列）仍然基于x86架构，但引入了许多新特性以提高性能和兼容性。

  2. ARM 指令集（精简指令集）：

    • 特点：ARM 指令集是一个精简指令集架构（RISC），其指令集较小而简单，旨在提供高效能和低功耗的处理能力。广泛用于移动设备和嵌入式系统中。
    • 历史背景：由Acorn Computers开发，ARM架构在1990年代成为主流，并不断演进（如ARMv7、ARMv8、ARMv9等）。
    • 现代化：ARM处理器被广泛应用于智能手机、平板电脑、嵌入式系统等设备，并且也开始进入高性能计算领域。

  3. MIPS 指令集（复杂指令集）：

    • 特点：MIPS 指令集也是一个CISC，最初在1990年代被广泛使用。它支持64位计算，设计用于高性能和高效率计算。
    • 历史背景：由MIPS Computer Systems开发，MIPS指令集在1990年代被认为是先进的，但由于市场因素和竞争，其影响力逐渐减弱。
    • 现代化：尽管MIPS在市场上有所退缩，但它仍在学术和特定应用中使用，并由中科院等机构维护和发展，具有自主知识产权。

  4. RISC-V 指令集（精简指令集）：

    • 特点：RISC-V 指令集是一个开源的RISC架构，旨在提供简单、灵活且可扩展的指令集。它可以适应各种计算需求，包括嵌入式系统、高性能计算和教育用途。
    • 历史背景：RISC-V项目由加州大学伯克利分校发起，设计理念强调开放性和简洁性，允许任何人使用和扩展。
    • 现代化：RISC-V正在得到越来越多的支持和应用，特别是在教育和开源硬件社区。许多公司和组织正在开发基于RISC-V的处理器和系统。

二：指令格式

1. 指令字长度

指令字长度是指在计算机指令中包含的二进制代码的位数。它决定了指令能够包含多少信息。计算机的机器字长决定了计算机能直接处理的二进制数据的位数（例如，64位操作系统和32位操作系统）。

  • 单字长指令：指令字长度等于机器字长度。例如，如果计算机的机器字长为32位，那么单字长指令的长度也是32位。

  • 半字长指令：指令字长度等于机器字长度的一半。例如，若机器字长为32位，则半字长指令的长度为16位。

  • 双字长指令：指令字长度等于机器字长度的两倍。例如，若机器字长为32位，则双字长指令的长度为64位。

  • 多字长指令：指令字长度等于几个机器字长度。例如，若机器字长为32位，则多字长指令的长度可能为128位或更多。

使用多字长指令的目的是提供足够的地址位来解决访问内存任意单元的寻址问题。然而，其主要缺点是需要两次或多次访问内存才能取出整条指令，从而降低了运算速度并占用了更多的存储空间。

2. 操作码

操作码（Opcode）是指令中的一部分，用于指定计算机应执行的操作。例如，操作码可以表示加法、减法、乘法等不同的操作。

  • 固定长度的操作码：所有指令的操作码长度相同。这种设计简单，控制容易，速度快，适用于指令条数较少的情况。

  • 可变长度的操作码：频繁使用的指令使用较短的操作码，不常用的指令则使用较长的操作码。这种设计提供了灵活性，但可能增加了解码的复杂度。

指令助记符（Mnemonic）是操作码的文本表示，例如ADD表示加法操作。这些助记符使编程更直观，但实际的机器指令仍由操作码来表示。

3. 地址码

地址码是指令中用于指定操作数位置的部分。根据指令中的地址数量，指令可以分为以下几类：

  • 三地址指令：

    • 特点：有三个地址码字段，通常用于指定两个操作数和一个操作结果的位置。三地址指令可以支持复杂的操作。
    • 例子：
      • ADD A1, A2, A3：将操作数A2和A3相加，结果存储到地址A1。
    • 存储器访问：如果A1、A2、A3都是内存单元，则需要四次访问存储器，这可能导致效率低下。

  • 二地址指令：

    • 特点：有两个地址码字段，一个用于源操作数，一个用于目的操作数。常见于现代计算机中。
    • 例子：
      • ADD R0, R3（RR型）：将寄存器R3中的数据加到寄存器R0中。
      • ADD R0, 100H（RS型）：将内存地址100H中的数据加到寄存器R0中。
      • ADD 100H, 200H（SS型）：将内存地址100H和200H中的数据相加，通常不推荐，因为访问内存的次数多，效率低下。
    • 转换示例：
      • MOV R0, 200H
      • ADD 100H, R0

  • 一地址指令：

    • 特点：有一个地址码字段，通常用于指定一个操作数，而操作结果存储在一个特定的寄存器中（如累加寄存器AC）。
    • 例子：
      • ADD 100H：将内存地址100H中的数据加到累加寄存器AC中，结果仍存储在AC中。

  • 零地址指令：

    • 特点：指令字中只有操作码，没有地址码。通常用于不需要操作数的操作，如停机指令。
    • 例子：
      • HALT：停机指令，不需要任何地址码。

三：指令和数据的寻址方式

1. 指令寻址

顺序寻址：
顺序寻址是指计算机执行程序时，通常一条指令接一条指令按顺序执行。程序计数器（PC，Program Counter）用于跟踪下一条要执行的指令的地址。每执行一条指令，PC的值会自动增加，指向下一条指令的地址。

  • 程序计数器（PC）：在顺序执行过程中，PC的值自动增加，以指向下一条指令。程序计数器的作用是追踪当前执行的指令位置。

跳跃寻址：
跳跃寻址用于执行循环或跳转指令时。程序的执行不会按照顺序进行，而是跳转到指定的目标地址。跳跃寻址时，指令会将目标地址直接装入PC中，改变程序执行的流程。

  • 跳转指令：例如条件跳转指令（如 JMP, CALL）用于改变程序的控制流。

2. 操作数寻址

立即寻址：
立即寻址方式将操作数直接嵌入到指令中，而不是给出操作数的地址。形式地址即为操作数本身。

  • 特征位：#
  • 例子：MOV R0, #10H 直接将值10H加载到寄存器R0中。

直接寻址：
直接寻址方式在指令中直接给出操作数的内存地址。操作数的地址在指令中明确指定，不需要额外计算。

  • 特征位：无
  • 例子：MOV R0, 10H 将内存地址10H中的内容加载到寄存器R0中。

间接寻址：
在间接寻址方式中，指令中给出的地址是一个内存位置，该内存位置存储了实际操作数的有效地址。因此，操作数的实际地址需要额外的内存访问来确定。

  • 特征位：无
  • 例子：MOV R0, (R1) 其中寄存器R1中存储了实际操作数的内存地址，指令从这个地址读取操作数。

寄存器寻址：
寄存器寻址方式将操作数存储在CPU的寄存器中，指令中指定寄存器的编号而不是内存地址。寄存器寻址速度快，但寄存器数量有限。

  • 特征位：Ri （寄存器编号）
  • 例子：MOV R0, R1 将寄存器R1中的内容移动到寄存器R0中。

寄存器间接寻址：
寄存器间接寻址方式中，寄存器中的内容是实际操作数的有效地址。寄存器的内容指向存储实际操作数的内存位置。

  • 特征位：@ 或 ()
  • 例子：MOV R0, @R1 或 MOV R0, (R1) 将寄存器R1中的地址内容中的数据加载到寄存器R0中。

偏移寻址：
偏移寻址方式将程序计数器（PC）的值与指令中给出的偏移量相加，以计算操作数的有效地址。用于相对位置的寻址。

  • 特征位：+
  • 例子：MOV R0, @(PC + 100H) 将相对于当前指令位置偏移100H的内存地址中的数据加载到寄存器R0中。

3. 高级寻址方式

相对寻址：
相对寻址方式是将程序计数器（PC）的当前值与指令中的偏移量相加，得到操作数的有效地址。偏移量通常用于实现跳转和分支操作。

  • 特征位：偏移量A
  • 例子：JMP 100H 将程序跳转到当前PC地址加上100H的位置。

基址寻址：
基址寻址方式使用一个基址寄存器的内容与指令中给出的形式地址相加，形成操作数的有效地址。适用于在较大的存储空间中寻址。

  • 特征位：基址寄存器内容 + 形式地址
  • 例子：MOV R0, [BR + 100H] 其中BR是基址寄存器。

变址寻址：
变址寻址方式将某个变址寄存器的内容与指令中给出的形式地址相加，以计算操作数的有效地址。变址寄存器的内容通常在程序运行过程中会变化。

  • 特征位：变址寄存器内容 + 形式地址
  • 例子：MOV R0, [IX + 100H] 其中IX是变址寄存器，100H是形式地址。

1. 立即寻址

例子：将立即数0x1A加载到寄存器R1中。

说明：指令中直接包含操作数0x1A，MOV指令将这个值立即加载到寄存器R1中。

2. 直接寻址

例子：将内存地址0x1000中的值加载到寄存器R2中。

说明：指令中给出内存地址0x1000，MOV指令将该地址中的值加载到寄存器R2中。

3. 间接寻址

例子：将寄存器R3中存储的地址的内容加载到寄存器R4中。

说明：寄存器R3中存储的是有效地址。MOV指令将这个地址中存储的值加载到寄存器R4中。

4. 寄存器寻址

例子：将寄存器R5中的值加载到寄存器R6中。

说明：MOV指令将寄存器R5中的值直接加载到寄存器R6中。

5. 寄存器间接寻址

例子：将寄存器R7中存储的地址的内容加载到寄存器R8中。

说明：寄存器R7中存储的是实际操作数的地址。MOV指令从这个地址读取数据，存储到寄存器R8中。

6. 偏移寻址

例子：将程序计数器（PC）的内容加上偏移量0x20的内存位置中的值加载到寄存器R9中。

说明：偏移量0x20相对于当前PC的值，计算得到操作数的有效地址，从该地址中读取数据到寄存器R9中。

7. 基址寻址

例子：将基址寄存器BR的内容加上偏移量0x30的内存地址中的值加载到寄存器R10中。

说明：BR寄存器中存储了基地址，偏移量0x30与基地址相加，得到实际的内存地址，从中读取数据到寄存器R10中。

8. 变址寻址

例子：将变址寄存器IX的内容加上偏移量0x40的内存地址中的值加载到寄存器R11中。

说明：IX寄存器中存储了变址值，偏移量0x40与变址值相加，得到实际的内存地址，从中读取数据到寄存器R11中。
