计算机组成原理之IO系统

I/O系统基本概念

I/O系统演变过程

I/O（输入/输出）系统是计算机系统中用于与外部设备进行数据交换的部分。其演变过程可以分为几个阶段，每个阶段都涉及到技术的进步和系统性能的提升：

  1. 早期阶段:

    • 手动操作: 最早的计算机系统需要人工手动进行数据输入和输出操作，例如通过打孔卡片或纸带。
    • 简单接口: 早期计算机接口简单，主要包括基本的键盘、打印机等。

  2. 程序控制I/O:

    • 中断驱动I/O: 引入了中断机制，使得I/O操作可以在设备准备好数据时中断CPU的执行，从而提高了效率。
    • 直接存储器访问（DMA）: DMA技术允许外设直接与内存进行数据交换，减少了CPU的负担。

  3. 通道控制I/O:

    • 通道机制: 使用通道控制器来管理复杂的I/O操作。通道控制器可以处理多个I/O设备的请求，提供了更高的效率和灵活性。
    • 多通道系统: 支持多个通道同时运行，提高了系统的I/O性能。

  4. 现代I/O系统:

    • 高速总线和接口: 现代计算机系统采用了高速总线（如PCIe、USB 3.0）和接口技术，支持更高的数据传输速率。
    • 智能I/O设备: 设备具有更高的智能化，例如网络接口卡、固态硬盘等，能够进行复杂的处理和自我管理。

I/O系统的基本组成

I/O系统的基本组成可以分为软件和硬件两个部分：

I/O软件

  1. 驱动程序:

    • 定义: 驱动程序是操作系统的一部分，用于控制和管理I/O设备。
    • 功能: 负责将操作系统和设备之间的命令和数据进行转换，使设备能够理解和执行操作系统的指令。

  2. 用户程序:

    • 定义: 用户程序是由应用程序编写的，用于请求和使用I/O服务。
    • 功能: 调用系统提供的I/O服务接口来进行数据输入输出操作。

  3. 管理程序:

    • 定义: 管理程序是操作系统中的一部分，负责管理I/O资源和调度。
    • 功能: 包括I/O调度、资源分配、错误处理等。

  4. 升级补丁:

    • 定义: 软件的修复和更新程序。
    • 功能: 修复I/O系统中的漏洞或功能缺陷，提升系统的性能和稳定性。
I/O硬件

  1. 外部设备:

    • 定义: 计算机系统中用于输入和输出的物理设备。
    • 例子: 键盘、鼠标、显示器、打印机、硬盘、网络接口等。

  2. 设备控制器:

    • 定义: 设备控制器是连接计算机与外部设备的硬件组件。
    • 功能: 负责控制设备的操作，处理数据的传输，转换设备的操作指令和计算机系统的指令。

  3. I/O接口:

    • 定义: I/O接口是计算机系统与外部设备之间的交接部分。
    • 功能: 作为连接计算机和外部设备的中介，负责数据的传输和设备的控制。
    • 类型:
      • 并行接口: 同时传输多位数据的接口，如打印机接口（LPT）。
      • 串行接口: 一次传输一位数据的接口，如RS-232串口。
      • USB接口: 通用串行总线接口，支持热插拔和即插即用功能。
      • 网络接口: 用于网络通信的接口，如以太网接口。
I/O指令与通道指令

  1. I/O指令:

    • 定义: 处理器直接执行的指令，用于进行I/O操作。
    • 功能: 用于启动设备、读取数据、写入数据等。

  2. 通道指令:

    • 定义: 专门用于通道控制器的指令，通常存储在主存中，由通道控制器执行。
    • 功能: 指定数据的首地址、传送字数、操作命令等，用于管理I/O设备的复杂操作。
通道控制

  • 通道机制:

    • 定义: 通道是用于管理和控制I/O操作的硬件组件。
    • 功能: 通过通道控制器来处理I/O操作，减少CPU的负担，提高系统的效率。

  • 通道指令的执行:

    • CPU: 启动I/O设备的指令。
    • 通道: 执行I/O操作的具体管理，负责与设备进行数据交换。

外部设备

外部设备，也称为外围设备，是计算机系统中除了主机以外的，能直接或间接与计算机交换信息的装置。外部设备通常分为输入设备、输出设备和外存储器。

输入设备

输入设备用于向计算机系统输入命令、文本、数据等信息。最基本的输入设备包括键盘和鼠标。

键盘

键盘用于输入字符和命令，其输入过程分为以下三个步骤：

  1. 识别按键：查出按下的是哪个键。
  2. 编码转换：将该键翻译成主机可以接收的编码，如ASCII码。
  3. 信息传输：将编码传送给主机。

鼠标

鼠标是常用的定位输入设备，其工作原理如下：

  • 机械式鼠标：通过滚轮和传感器检测运动。
  • 光电式鼠标：利用光学传感器检测运动方向和距离，控制光标在屏幕上的运动。

输出设备

输出设备用于将计算机中的信息输出到计算机外部进行显示或交换。最基本的输出设备包括显示器和打印机。

显示器

显示器用于显示计算机输出的信息，其主要参数包括：

  • 屏幕大小：以对角线长度表示，常用的有12~29英寸等。
  • 分辨率：显示器所能显示的像素个数。例如，800×600、1024×768和1280×1024等。
  • 灰度级：黑白显示器中像素点的亮暗差别，彩色显示器中表现为颜色的不同。常见的有8位（256级）、16位等。
  • 刷新率：单位时间内扫描整个屏幕的次数。通常刷新频率在60~120Hz。
显示器类型
  • 阴极射线管 (CRT) 显示器：由电子枪、偏转线圈、荫罩、高压石墨电极和荧光粉涂层组成。优点包括可视角度大、色彩还原度高等。
  • 液晶显示器 (LCD)：利用液晶的电光效应，由薄膜晶体管控制液晶分子的光学特性来显示图像。特点包括体积小、重量轻、省电、无辐射。
  • LED 显示器：通过控制半导体发光二极管显示信息，具有高亮度、低功耗等优点。
显示器分类
  • 字符显示器：以点阵为基础进行字符显示。点阵决定了显示字符的质量和窗口大小。
  • 图形显示器：通过显际文件和缓冲存储器显示图形，适合高分辨率和曲线平滑的图形显示。
  • 图像显示器：主要用于显示图像，通常为高质量显示器。

打印机

打印机用于将计算机输出的信息打印到纸上，主要分为击打式和非击打式打印机。

击打式打印机
  • 针式打印机：通过针头在色带和纸上打出字符。优点是成本低、打印质量好；缺点是噪声大、速度慢。
非击打式打印机
  • 喷式打印机：通过带电墨滴在纸上形成字形，通常速度较快、噪声小，但防水性差。
  • 激光打印机：使用激光束扫描感光鼓，经过显影、转印和定影打印字符或图像。具有高质量、高速度、低噪声的优点。

外存储器

外存储器指计算机内存和CPU缓存之外的存储器。外存储器包括硬磁盘、光盘、固态硬盘等，既可以作为输入设备，也可以作为输出设备。

磁盘存储器

  • 组成：包括磁盘驱动器、磁盘控制器和盘片。磁盘由若干个记录面、磁道和扇区组成。

  • 性能指标：

    • 磁盘容量：包括非格式化容量和格式化容量。
    • 记录密度：道密度、位密度和面密度。
    • 平均存取时间：寻道时间、旋转延迟时间和传输时间。
    • 数据传输率：单位时间内的数据传输量。

  • 磁盘阵列：RAID技术将多个物理磁盘组合成一个逻辑磁盘，提高存储性能和可靠性。

    • RAID0：无冗余，速度快。
    • RAID1：镜像，数据冗余。
    • RAID5：奇偶校验，提供容错能力。

光盘存储器

光盘存储器使用光学原理读写信息，常见类型有：

  • CD-ROM：只读光盘。
  • CD-R：可写入一次。
  • CD-RW：可读写。
  • DVD-ROM：高容量的光盘。

固态硬盘

固态硬盘使用Flash Memory作为存储介质，具有快速读写、高耐用性和低功耗等优点。

I/O接口 (I/O 控制器)

I/O接口（也称为I/O控制器）是主机和外设之间的信息交换界面。它起到了主机系统和外部设备之间的桥梁作用，负责协调和管理数据的传输。

I/O接口的功能

  1. 设备选址：分配并管理每个I/O设备的地址。
  2. 传送命令：将来自主机的命令发送到指定的I/O设备。
  3. 传送数据：在主机和I/O设备之间传输数据。
  4. 反映I/O设备的工作状态：提供I/O设备的状态信息，如是否准备好进行数据交换。

I/O基本结构

I/O接口的基本结构包括以下几个主要部分：

  • 地址寄存器：用于存储I/O设备的地址。
  • 数据寄存器：用于存储要发送或接收的数据。
  • 控制寄存器：用于发送控制命令，例如读写操作的指示。
  • 状态寄存器：用于读取I/O设备的状态信息，如是否忙碌或故障。

I/O端口及其编址

I/O端口是指可以直接被CPU访问的寄存器。I/O端口的编址方式主要有以下两种：

统一编址（存储器映射方式）：

  • 定义：将I/O端口当作内存单元进行地址分配。
  • 优点：
    • 不需要专门的输入/输出指令。
    • CPU对I/O操作更加灵活。
    • 端口编址空间较大。
  • 缺点：
    • 占用内存地址空间，减少可用内存容量。
    • 数据输入/输出操作执行速度较慢。

独立编址（I/O映射方式）：

  • 定义：I/O端口的地址与内存地址独立，使用专门的I/O指令访问端口。
  • 优点：
    • 输入/输出指令与存储器指令明显区分，程序编写清晰。
    • 控制信号更专一，易于理解。
  • 缺点：
    • 输入/输出指令集较少。
    • 需要CPU提供存储器读/写和I/O设备读/写两组控制信号，增加了控制复杂性。

I/O接口的类型

按数据传送方式：

  • 并行接口：

    • 定义：一个字节或一个字的所有位同时传送。
    • 特点：传输速度较快，但需要多条线路，复杂度高。

  • 串行接口：

    • 定义：一位一位地传送数据。
    • 特点：传输线路较少，成本低，但传输速度较慢。

注：在主机和接口之间的数据传输通常是并行的，接口负责完成数据格式的转换。

按主机访问I/O设备的控制方式：

  • 程序查询接口：

    • 定义：主机通过程序周期性地查询I/O设备的状态。
    • 特点：简单实现，但效率较低。

  • 中断接口：

    • 定义：I/O设备在完成操作或发生事件时，主动向主机发出中断信号。
    • 特点：可以高效地处理异步事件，但需要中断处理程序和中断管理机制。

  • DMA接口（直接内存存取）：

    • 定义：I/O设备可以直接与内存交换数据，不通过CPU。
    • 特点：提高数据传输效率，减少CPU负担。

按功能选择的灵活性：

  • 可编程接口：

    • 定义：接口的操作和功能可以通过编程设置和调整。
    • 特点：灵活性高，可以适应不同的需求。

  • 不可编程接口：

    • 定义：接口的功能和操作是固定的，不可更改。
    • 特点：简单易用，但灵活性低。

I/O方式

I/O方式指的是主机与I/O设备之间数据传输的不同方法。主要包括程序查询方式、程序中断方式和DMA方式。

程序查询方式

程序查询方式通过不断查询I/O设备的状态来确定是否需要进行数据传输。CPU执行程序时会定期检查I/O设备的状态寄存器，以确定设备是否准备好进行数据传输。

  • 流程图：展示了程序查询方式的基本操作步骤，包括查询状态、判断设备是否准备好、进行数据传输等。

  • 一次传送一个字：在程序查询方式中，每次数据传输的单位通常是一个字节或一个字。数据在传送过程中被存放在CPU寄存器中。

  • 接口结构：程序查询方式的接口结构通常包括状态寄存器、数据寄存器和控制寄存器等。

程序中断方式

程序中断方式允许I/O设备在需要处理数据时向CPU发出中断信号，CPU会暂停当前的程序，处理I/O设备的请求，然后再返回到中断发生时的程序位置继续执行。

4.2.1 中断系统

中断的基本概念：
程序中断是指在计算机执行现行程序的过程中，出现某些急需处理的异常情况或特殊请求时，CPU暂时中止现行程序，处理这些情况后再返回继续执行原程序。

中断分类：

  • 硬件中断：由硬件设备发出的中断信号。
  • 软件中断：由程序执行中的特殊指令触发的中断。

中断请求标记：

  • 每个中断源向CPU发出中断请求时，使用中断请求标记触发器（INTR）。
  • 当INTR状态为“1”时，表示中断源有请求。
  • CPU响应中断请求的时机是在每条指令执行阶段结束前。
  • 中断响应条件：中断源有请求、CPU允许中断、指令执行完毕且无更紧迫任务。

工作流程：

  1. 中断请求：中断源向CPU发送中断请求信号。
  2. 中断响应：CPU响应中断请求。
  3. 中断判优：当多个中断源同时请求时，使用硬件或软件判优机制。
    • 硬件判优：通过硬件排队器实现。
    • 软件判优：通过查询程序实现。
  4. 中断处理：
    • 进入中断服务程序：将中断服务程序的地址放入程序计数器（PC）。
    • 保护现场：保存程序断点和寄存器内容。
    • 中断服务：执行中断服务程序。
    • 恢复现场：将保存的寄存器内容恢复到CPU中。
    • 中断返回：将程序计数器恢复到原程序的断点处。

单重中断与多重中断：

  • 单重中断：执行中断服务程序时不响应新的中断请求。
  • 多重中断：也称中断嵌套，执行中断服务程序时可以响应新的中断请求。

中断屏蔽技术：

  • 通过设置中断屏蔽字来控制中断请求的处理，屏蔽字寄存器中每个中断源有一个屏蔽触发器。

DMA方式

DMA（直接内存存取）方式允许外设直接与主存交换数据，而无需CPU干预。DMA控制器负责数据传输的全过程，从而提高了数据传输的效率。

  • DMA控制器功能：

    1. 接受外设的DMA请求，并向CPU发出总线请求。
    2. CPU响应总线请求，将控制权交给DMA控制器。
    3. 确定数据传送的主存单元地址及长度，自动修改计数器。
    4. 发出读写控制信号，执行数据传送操作。
    5. 向CPU报告DMA操作的结束。

  • 传送方式：

    • 停止CPU访问主存：在数据传送期间，CPU不访问主存，DMA控制器完全控制总线。
    • DMA与CPU交替访存：CPU周期分为DMA访问和CPU访问，不需要总线使用权的申请和归还过程。
    • 周期挪用：I/O设备在需要时占用一个或几个存取周期。

  • DMA访问主存的三种情况：

    1. CPU不访问主存（无冲突）。
    2. CPU正在访问主存（结束存取周期后让出总线）。
    3. CPU与DMA同时请求访问主存（I/O请求优先）。

DMA方式的特点：

  1. 使主存与CPU的联系脱钩，主存可以同时被CPU和外设访问。
  2. 硬件直接实现地址确定和数据计数。
  3. 需要主存中开辟专用缓冲区。
  4. 提高了数据传输速度，CPU和外设可以并行工作。
  5. DMA传送前需要通过程序进行预处理，传送后通过中断进行后处理。

与中断方式的区别：

  • 中断：程序控制，处理异常事件，响应时间为每条指令周期结束时。
  • DMA：硬件控制，仅进行数据传送，响应时间为每个机器周期结束时。
