计算机组成原理之数据的表示和运算

一、进位计数制

进位计数制是根据不同进制进行数字表示和运算的方式。常见的进制包括十进制、二进制、八进制、十六进制以及少见的六十进制。不同进制的存在是因为在不同的场景下，它们有独特的意义和应用场景：

  • 十进制：我们日常生活中的数学运算，使用的都是十进制。
  • 二进制：在计算机系统中，二进制是基础，因其与计算机的逻辑电路和存储方式相符。
  • 八进制和十六进制：在计算机领域，它们常用于简化表示二进制数据。
  • 六十进制：时间系统中的进制，如 60 秒等于 1 分钟。

八进制和十六进制的计算示例

不同进制的计算方式要特别小心，尤其是在满位进位时。以下是一些简单的例子来说明这一点：

  • 八进制运算：
    $5.4_8 + 1.4_8 = 7.0_8$

    • 解释：八进制满 8 进 1，小数部分 ( $4_8 + 4_8 = 10_{10}$ )，在八进制中等于 ($ 8_8 + 0_8 $)（即满 8 进 1，小数部分变成 ( $0_8$ )）。

  • 十六进制运算：
    $
    5.8_{16} + 0.9_{16} = 6.1_{16}$

    • 解释：十六进制满 16 进 1，小数部分 ( $9_{16} + 8_{16} = 17_{10} = 16_{16} + 1_{16} $)，所以小数部分为 ($ 1_{16}$ )，整数部分 ( $5_{16} + 1_{16} = 6_{16}$ )。

二进制到十六进制的转换

当我们处理较大的十进制数时，可以转换为二进制或十六进制来简化计算。以下是一个例子，展示如何将十进制数 75 转换为二进制：

  1. 首先，找到小于 75 的最大 2 的幂，64（即 ( $2^6$ )），因此在二进制的第 7 位为 1。

    • 剩下的部分：75 - 64 = 11。

  2. 接着，找到小于 11 的最大 2 的幂，8（即 ($ 2^3$ )），因此在二进制的第 4 位为 1。

    • 剩下的部分：11 - 8 = 3。

  3. 最后，找到小于 3 的最大 2 的幂，2（即 ( $2^1$ )），因此在二进制的第 2 位为 1，余下 1 在第 1 位为 1。

    • 最终结果：( $75_{10} = 1001011_2 $)。

小数部分的表示

在某些进制中，小数部分可能无法准确表示。例如，十进制中的 ( $0.3_{10}$ ) 在二进制中是无法用有限位表示的，因此通常需要截断或舍入，导致一些精度损失。

真值和机器数

  • 真值：指的是人类习惯使用的十进制数，如 ( 100 ) 或 ( -13 )。真值是我们理解和使用数字的方式，但它不能直接用于计算机存储和处理。

  • 机器数：指的是计算机能够识别和处理的数字形式。在机器数中，符号位通常用 0 表示正数，用 1 表示负数。计算机通过这种表示方式来区分正负数。

    例如：

    • ( $0 \, 1111 $) 表示正数 ( $15_{10} $)。
    • ( $1 \, 1000 $) 表示负数 ($ -8_{10}$ )（使用补码表示负数）。

详细示例：机器数的表示

对于一个 4 位机器数，使用补码来表示负数的规则如下：

  • 正数：直接用二进制表示。例如，( $5_{10} = 0101_2$ )。
  • 负数：需要将数取补码。例如，( $-5_{10} $) 的补码表示为：
    1. 将 ($ 5_{10}$ ) 的二进制形式 ($ 0101_2$ ) 取反为 ( $1010_2$ )。
    2. 末位加 1 得到 ( $1011_2$ )，即为 ($ -5_{10}$ ) 的补码表示。

补码的表示方式确保了计算机可以在二进制电路中统一处理加法和减法。

二、BCD码

BCD码（Binary-Coded Decimal，二进制编码的十进制）是用二进制来表示十进制数字的方法，每一个十进制数位都单独用一组二进制位来表示。常见的BCD码有8421码、余3码、2421码等。

1. 8421码

8421码是一种常见的BCD编码方式，使用 4 位二进制来表示一个十进制数位。8421码的名称来源于这 4 位二进制数各自的权值：( $2^3 = 8$ )、( $2^2 = 4$ )、( $2^1 = 2$ )、($2^0 = 1$ )。

例如：

  • ( $9_{10} = 1001_2$)（因为 ($8 + 1 = 9$ )）
  • ( $5_{10} = 0101_2$ )

8421码的表示冗余：
因为 4 位二进制可以表示 ( $2^4 = 16$ ) 个数，而十进制只有 10 个数位，因此8421码的二进制表示中有 6 种组合（从 ($ 1010_2$ ) 到 ( $1111_24$ )）是冗余的，不用于表示有效的十进制数。

例如，数字 ( $985_{10}$ ) 用 8421 码表示为：

  • ( $9_{10} = 1001_2$ )
  • ( $8_{10} = 1000_2 $)
  • ( $5_{10} = 0101_2$ )
    因此，985 用 8421 码表示为：1001 1000 0101。

8421码的加法运算
8421码的加法与十进制加法相似，但在某些情况下需要修正。当8421码的加法结果大于 9 时，必须加上 ($ 6_{10} $)（即 ($ 0110_2$ )）来修正结果。

例子：

  • 求 ( 5 + 8 ) 的 8421 码表示：
    • 方法一：转换成十进制，5 + 8 = 13。再将13转换为8421码：($ 1 \, 0011_2$ )。
    • 方法二：直接相加8421码：
      $0101_2 + 1000_2 = 1101_2$
      但是，( $1101_2$ ) 并不在有效的8421码范围内（8421码只表示 ( $0000_2$ ) 到 ($ 1001_2 $)）。因此需要加上 ($ 6_{10} $)（即 ( $0110_2 $)）来修正：
      $1101_2 + 0110_2 = 1 \, 0011_2$
      最终结果为 ( 1 \, 0011_2 )，即 ( 13_{10} )。

另一个例子：
求 ( 9 + 9 ) 的 8421 码表示：
$1001_2 + 1001_2 = 1 \, 0010_2$
这个结果还不正确，因为它落在了无效范围，需要加 ( 0110_2 ) 修正：
$1 \, 0010_2 + 0110_2 = 1 \, 1000_2$
最终结果为 ( $1 \, 1000_2$)，即 ($ 18_{10}$ )。

2. 余3码

余3码 是在8421码的基础上，每个十进制数位再加上3来表示。简单理解，就是用8421码表示十进制数位时，再加上 ( $0011_2$ ) （即 ($ 3_{10} $)），使每个数位的表示值加3。

例如：

  • ( $0_{10} $) 在8421码中为 ( $0000_2 $)，在余3码中为 ($ 0011_2 $)。
  • ( $9_{10}$ ) 在8421码中为 ( $1001_2$ )，在余3码中为 ($ 1100_2$ )。

余3码的一个特点是它不再是一个有权码，而是通过简单的位组合来表示。

3. 2421码

2421码 是一种改变了权值的有权码。它的4个二进制位的权值分别是2、4、2、1。不同于8421码中每一位的权值是固定的，2421码改变了位的权值分配。

例如：

  • ( $5_{10}$ ) 在2421码中的表示为 ( $0101_2$ )，因为 ( $0 \times 2^3 + 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0 = 5_{10} $)。
  1. BCD码的概念
    • 定义：二进制编码的十进制数
    • 应用：数字显示、金融系统、简单计算器等
  2. 8421码
    • 定义：有权码，权值为8、4、2、1
    • 运算规则：加法运算时需考虑加 ($ 0110_2 $) 来修正
    • 示例：985的8421码表示
  3. 余3码
    • 定义：在8421码基础上，每个数位加3
    • 特点：非有权码
    • 示例：数字9的余3码表示
  4. 2421码
    • 定义：有权码，权值为2、4、2、1
    • 特点：特殊的权值分配
    • 示例：数字5的2421码表示

三、字符与字符串

1. 英文字符

ASCII码 是一种通用的字符编码，用于表示英文字符、数字和控制字符。ASCII码使用 7位二进制 来表示常用字符，共有128个字符（($2^7 = 128$)）。但在存储时，通常使用1个字节（8位）来存储，因此最高位补零。

  • 可打印字符：ASCII码32到126的范围表示常见的可打印字符，如空格、标点符号、数字和字母。
  • 控制字符：ASCII码0到31的字符是控制字符，如回车（13）、换行（10）等。

数字和字母的ASCII编码规律：

  • 数字0-9的ASCII码：十进制值为48到57，二进制值为 ( $0011 \, 0000$ ) 到 ($ 0011 \, 1001$ )。
  • 大写字母‘A’的ASCII码是65（( $0100 \, 0001$ )），‘H’的ASCII码是72，二进制表示为 ( $0100 \, 1000 $)。

常见的题型：

  • 求字母的ASCII码：如已知‘A’的ASCII码是65，‘H’的ASCII码为72，那么‘H’的二进制表示为 ( 0100 \, 1000 )。

2. 中文符号

GB 2312-80 是1980年中国制定的标准，用来编码简体中文的常用字符（包括7445个汉字）。GB 2312使用 区位码 的概念来表示每个汉字，类似二维数组的行和列：

  • 区码 表示第几行。
  • 位码 表示列号。

国标码与机内码的差异：

  • 国标码：为避免与ASCII控制字符冲突，在区位码上加上32（即 ( 20H )），称为国标码。
  • 机内码：再加128（即 ( 80H )），使之与英文字符区分开，称为机内码。

输入编码与输出字形码：

  • 输入编码：如拼音、五笔编码，用于输入中文字符。
  • 输出码：是屏幕显示的汉字形码。

3. 字符串

字符串是由字符组成的序列，通常用一组字符组成的数据结构来表示。在大多数编程语言中，字符串以字符数组的形式存在，常见的字符串处理包括存储、修改和检索。

字符串的存储与终止符：

  • 在C语言中，字符串以 '\0'（空字符）作为结束符，表示字符串的终结。
  • 在其他语言（如Python、Java）中，字符串的长度是动态的，内存管理和存储方式由语言的运行时系统控制。

存储顺序：大端模式与小端模式

字符和数字在计算机中的存储顺序因系统架构不同而异，主要有 大端模式（Big Endian） 和 小端模式（Little Endian） 两种：

  • 大端模式（Big Endian）：数据的高位字节存储在内存的低地址，低位字节存储在高地址。这种存储方式符合人类的阅读习惯。例如，十六进制数0x12345678，在大端模式下的存储顺序为：

    • 地址0x00：12
    • 地址0x01：34
    • 地址0x02：56
    • 地址0x03：78

  • 小端模式（Little Endian）：数据的低位字节存储在内存的低地址，高位字节存储在高地址。这种方式与人类的习惯相反。例如，0x12345678 在小端模式下的存储顺序为：

    • 地址0x00：78
    • 地址0x01：56
    • 地址0x02：34
    • 地址0x03：12

大端和小端模式的区别与应用：

  • 大端模式 在一些网络协议（如TCP/IP）中被广泛使用，因为它与人类的自然阅读顺序一致。
  • 小端模式 则在大多数PC架构（如x86处理器）中应用较多，因为它可以提高数据处理的效率。

四、校验码

在数据传输中，为了确保数据的完整性，校验码被广泛应用。校验码通过在数据中附加一些特定的位来检测或纠正传输中的错误。常见的校验码类型包括奇偶校验码、海明码和循环冗余校验（CRC）。

1. 奇偶校验码

奇偶校验码是一种简单的错误检测方法。它通过增加一个校验位来确保数据位中“1”的个数为奇数或偶数，以此判断数据是否出错。

奇偶校验的组成：

  • n 位有效信息位：包含实际的数据内容。
  • 1 位校验位：根据信息位中的1的个数确定。

奇偶校验的规则：

  • 奇校验码：数据中的1的个数必须是奇数。如果1的个数为偶数，校验位设为1，否则为0。
  • 偶校验码：数据中的1的个数必须是偶数。如果1的个数为奇数，校验位设为1，否则为0。

例子：
假设数据位为 1100110，我们来计算它的奇偶校验位。

  • 对于奇校验，1100110 中有 4 个 1（偶数），因此校验位设为 1，完整码为 11001101。
  • 对于偶校验，校验位设为 0，完整码为 11001100。

奇偶校验能有效地检测出奇数数量的错误位变动，但如果位的跳变数为偶数（例如两个位同时出错），则无法检测出错误。

概念补充：

  • 码字：由若干位组成的一个完整的数据段。
  • 码字间的距离 (d)：两个码字逐位对比，不同的位的个数。
  • 码距：若干合法码字中，任意两个码字之间的最小距离。

检错和纠错能力：

  • (d = 1)：无检错能力。
  • (d = 2)：具有检错能力。
  • ($d \geq 3$)：合理设计可以同时具备检错和纠错能力。

海明码是一种能够同时进行检错和纠错的校验码。其特点是可以检测两位错误并纠正一位错误。虽然概念较为复杂，但其结构和原理非常实用。

3. 循环冗余码（CRC）

循环冗余校验码（CRC）是一种常用于数据链路层的校验码，能够通过“除法”方法检测数据错误。

CRC原理：

  • 数据发送方和接收方预先约定一个除数，数据加上校验位后进行除法运算，要求余数为 0。
  • 发送方将数据和除数相除后，计算得到余数并附加在数据后发送。接收方同样用除数对接收的数据进行除法运算，如果余数不为 0，则说明数据出错。

例子：
假设信息位为 101001，除数为 1101，我们首先将信息位后补 3 个 0（因除数为 3 位数），形成 101001000，然后进行二进制除法，得到余数 001。将余数加在信息位后，形成完整的 CRC 码 101001001。

计算步骤：

  1. 信息位 101001 补 0 后为 101001000。
  2. 使用除数 1101 对其进行二进制除法，余数为 001。
  3. 最终的 CRC 码为 101001001。
  • 奇偶校验码适用于简单的检错，特别是单位错误，但对偶数位错误无效。
  • 海明码能够检测并纠正单位错误，并且适合用于需要较高可靠性的场景。
  • CRC虽然不具备纠错功能，但其检错效率高，适用于数据通信中的错误检测。

这些校验码在计算机系统和通信协议中广泛应用，确保数据在传输过程中不受错误的影响。

五、定点数的表示与运算

1. 定点数的表示

（一）无符号数

无符号数是最简单的二进制数表示方式，只能表示非负整数。每一位都用于表示数值，因此可以表示的范围是从 (0) 到 ($2^n - 1$)，其中 (n) 是位数。例如：

  • 8位无符号数：
    $
    \text{范围} = 0 \text{ 到 } 255
    $

  • 16位无符号数：
    $
    \text{范围} = 0 \text{ 到 } 65535
    $

（二）有符号数

有符号数可以表示正数和负数。常用的表示方法有：

  • 原码
    在原码表示中，最高位（符号位）用于表示符号，其余位用于表示数值。符号位为 0 表示正数，为 1 表示负数。

    • 正数：其原码与无符号表示相同。例如，+5 的 8 位原码表示为 00000101。
    • 负数：其原码是绝对值的二进制表示，加上符号位。例如，-5 的 8 位原码表示为 10000101。

  • 反码
    反码的表示方法是在原码的基础上对负数的所有位（除了符号位）取反。

    • 正数：其反码与原码相同。
    • 负数：将其原码中除符号位外的每一位取反。例如，-5 的反码是 11111010（原码 10000101 取反）。

  • 补码
    补码是负数的反码加1，使得加法和减法可以统一处理，避免了原码和反码的复杂问题。

    • 正数：其补码与原码相同。
    • 负数：取其反码，然后加1。例如，-5 的补码是 11111011（反码 11111010 加 1）。

  • 移码
    移码（或偏移码）是将原码的数值加上一个固定的偏移量。常用于浮点数的指数部分，以便于表示负数和简化计算。偏移量是 ($2^{(n-1)}$)，其中 (n) 是位数。

    • 8位移码：偏移量为 ($2^7 = 128$)。
    • 表示范围：-128 到 127。例如，+5 的移码表示为 00000101，-5 的移码表示为 11111011（即 (5 + 128)）。

      • 原码：00000010
      • 补码：00000010
    • 相加：
      $
      11111101 \text{ (即 -3)} + 00000010 \text{ (即 2)} = 11111111 \text{ (即 -1)}
      $

  • 计算 5 - 3：

    • 5 的补码：
      • 原码：00000101
      • 补码：00000101
    • -3 的补码：11111101
    • 相加：
      $
      00000101 \text{ (即 5)} + 11111101 \text{ (即 -3)} = 00000000 \text{ (即 2)}
      $

2. 各种码的作用

1. 原码（Sign-Magnitude）

  • 作用：原码是一种直观的数值表示方式，用于表示正数和负数。其最高位（符号位）明确地指示了数值的符号（0 为正，1 为负），其余位用于表示数值的绝对值。
  • 优点：
    • 直观：对人类来说，原码的符号位直接反映了数字的符号。
    • 简单：符号位的分离使得理解数值的符号变得简单。
  • 缺点：
    • 加法复杂：原码加法需要对符号位进行单独处理，计算时的处理过程比较复杂。
    • 存在两个零：正零和负零的存在增加了处理的复杂性。

2. 反码（One’s Complement）

  • 作用：反码是通过将原码中的数值位取反（0 变 1，1 变 0）来表示负数。它简化了负数的表示，并使得减法操作较原码更加简便。
  • 优点：
    • 简化负数表示：负数的反码是将正数的每一位取反，计算时相对简单。
    • 减法处理简化：在反码表示中，减法操作可以转化为加法，简化了计算过程。
  • 缺点：
    • 存在两个零：同样存在正零和负零的问题。
    • 溢出问题：加法运算可能会导致溢出，且需要额外处理进位问题（进位需要加到结果中）。

3. 补码（Two’s Complement）

  • 作用：补码是对负数进行反码取反后加1的结果。它将加法和减法操作统一处理，简化了计算机的硬件设计。
  • 优点：
    • 简化计算：补码表示使得加法和减法操作统一，无需特殊处理符号位。
    • 避免两个零问题：只有一个零，避免了正零和负零的问题。
    • 溢出检测：溢出检测和处理更加直接，只需检查符号位的变化。
  • 缺点：
    • 表示范围限制：补码的表示范围是有上限的，例如 8 位补码的范围是 -128 到 127。

4. 移码（Excess-N）

  • 作用：移码是一种通过加上固定偏移量的方式表示数值的方法（补码的基础上将符号位取反）。
  • 优点：
    • 简化比较：使得指数部分可以用直接的二进制数值进行比较，无需考虑符号位。
    • 表示范围：通过偏移量，可以有效地表示更大的范围。
  • 缺点：
    • 额外计算：需要在计算时减去偏移量来恢复实际的指数值。
    • 偏移量选择：偏移量的选择影响到表示的范围和精度。

3. 移位运算

移位运算是一种在二进制数中移动位的操作，常用于计算机系统中。移位运算有多种形式，包括算术移位、逻辑移位和循环移位。

（i）算术移位

  • 定义：算术移位是在进行位移时，保持符号位不变。移位操作会将数值向左或向右移动，同时处理符号位的影响。
  • 左移操作：将数值的所有位向左移动，最低位填充0。左移相当于将数值乘以2。例如：
    • 数值 00000101（十进制 5），左移一位得到 00001010（十进制 10）。

  • 右移操作：将数值的所有位向右移动，符号位（最高位）不变。右移相当于将数值除以2（向下取整）。例如：

    • 数值 11111010（十进制 -6，使用补码表示），右移一位得到 11111101（十进制 -3）。

  • 示例：

    • 左移：00001101 << 1 = 00011010 (13 左移 1 位变为 26)
    • 右移：00001101 >> 1 = 00000110 (13 右移 1 位变为 6)
    • 负数右移（算术右移）：11101100 >> 1 = 11110110 (负数 -20 右移 1 位变为 -10)

（ii）逻辑移位

  • 定义：逻辑移位是在进行位移时，符号位不考虑，所有移出的位都被填充为0。逻辑移位适用于无符号数。
  • 左移操作：将数值的所有位向左移动，最低位填充0。左移相当于将数值乘以2。例如：
    • 数值 00000101（十进制 5），左移一位得到 00001010（十进制 10）。

  • 右移操作：将数值的所有位向右移动，最高位填充0。右移相当于将数值除以2（向下取整）。例如：

    • 数值 00000101（十进制 5），右移一位得到 00000010（十进制 2）。

  • 示例：

    • 左移：00001101 << 1 = 00011010 (13 左移 1 位变为 26)
    • 右移：00001101 >> 1 = 00000110 (13 右移 1 位变为 6)
    • 负数逻辑右移（逻辑右移）：11101100 >> 1 = 01110110 (负数 -20 右移 1 位变为 118)

（iii）循环移位

  • 定义：循环移位是一种特殊的移位操作，其中移出的位会重新插入到另一端。循环移位有两种主要形式：左循环移位和右循环移位。
  • 左循环移位：将数值的所有位向左移动，移出的位会重新插入到最低位。例如：
    • 数值 11010010（二进制），左循环移位一位得到 10100101。

  • 右循环移位：将数值的所有位向右移动，移出的位会重新插入到最高位。例如：

    • 数值 11010010（二进制），右循环移位一位得到 01101001。

  • 示例：

    • 左循环移位：11010010（原数），左循环移位一位得到 10100101（最高位移到最低位）
    • 右循环移位：11010010（原数），右循环移位一位得到 01101001（最低位移到最高位）
  • 算术移位：保持符号位不变，适用于有符号数。左移相当于乘以2，右移相当于除以2（向下取整）。
  • 逻辑移位：不考虑符号位，适用于无符号数。左移相当于乘以2，右移相当于除以2（向下取整），移出的位用0填充。
  • 循环移位：移出的位重新插入到另一端，无论是左循环还是右循环，都会将位移动到另一端，形成循环效果。

4. 加减运算和溢出判断

1. 加法

加法操作是在进行数值计算时最基本的操作之一。不同编码下的加法有不同的处理方式：

  • 无符号数加法：

    • 直接对两个数进行二进制加法。
    • 如果结果超出了表示范围（例如 8 位无符号数的范围是 0 到 255），则会产生溢出。例如，255 + 1 的结果是 256，超出了 8 位的范围，结果会变为 0。

  • 有符号数加法（补码）：

    • 直接对两个补码数进行加法。
    • 对于补码，加法操作可以简化为无符号数加法。结果会自动处理负数的情况。
    • 溢出判断需要关注符号位的变化。具体步骤如下：
      • 正数加正数：若结果为负数，说明溢出。
      • 负数加负数：若结果为正数，说明溢出。
      • 正数加负数：一般情况下不发生溢出，结果符合预期。

  • 加法溢出判断：

    • 无符号数：溢出检测通过查看最高位的进位是否超出范围来确定。如果最高位进位存在，则说明发生溢出。
    • 有符号数（补码）：通过符号位的变化来判断。具体方法是：
      • A和B都是正数：如果结果为负数，则发生溢出。
      • A和B都是负数：如果结果为正数，则发生溢出。
      • A和B符号不同：无论结果是什么，都不发生溢出。

2. 减法

减法可以转换为加法操作。对于补码表示的减法，减法操作可以通过加法来实现：

  • 补码减法：

    • A - B 可以转换为 A + (-B)。其中 -B 是 B 的补码。
    • 实际操作中，将 B 取反后加1，然后与 A 相加。
    • 对于补码，减法操作与加法在处理上没有区别。

  • 减法的溢出判断：

    • A - B 通过将 B 取反加1转化为 A + (-B)。
    • 依据加法的溢出规则进行判断。结果若出现异常符号位变化，即表明发生溢出。

3. 溢出判断

溢出是指计算结果超出了表示数值的范围。在不同的数值表示方法中，溢出判断有所不同：

  • 无符号数：

    • 溢出判断：通过观察计算结果的最高位进位来判断。如果最高位有进位，则发生溢出。

  • 有符号数（补码）：

    • 加法：
      • 如果两个操作数的符号位相同，但结果的符号位不同，则发生溢出。例如：
        • 正数加正数结果为负数。
        • 负数加负数结果为正数。
    • 减法：
      • 转换为加法后，用加法的溢出规则判断。如果结果的符号位不符合预期，则说明发生溢出。

  • 溢出示例：

    • 无符号加法：1111 (15) + 0001 (1) = 0000 (0) (8 位无符号数，溢出)
    • 补码加法：0110 (6) + 0110 (6) = 1100 (-4) (8 位补码数，溢出)
  • 加法：在有符号数（补码）下，加法可以简化为无符号加法。溢出判断依据符号位变化。
  • 减法：通过取反加1的方式转化为加法进行计算。溢出判断可以参考加法的溢出规则。
  • 溢出判断：无符号数溢出通过进位位判断，有符号数（补码）通过符号位变化判断。

5. 原码的乘法运算

原码的乘法运算涉及到逐位相乘并累加的过程。由于原码的表示方式相对复杂，特别是涉及符号位时，乘法运算的步骤包括以下几个部分：

1. 原码表示

  • 正数：原码表示与其无符号二进制表示相同。例如，+5 的原码是 00000101（假设用 8 位表示）。
  • 负数：负数的原码是其绝对值的二进制表示，最高位（符号位）为 1。例如，-5 的原码是 10000101（假设用 8 位表示）。

2. 乘法步骤

乘法运算的基本方法是逐位相乘，然后累加结果。具体步骤如下：

  1. 处理符号位：

    • 判断两个数的符号位（最高位），用来确定结果的符号。
    • 符号位处理后，将原码转换为绝对值的二进制数进行乘法运算。

  2. 逐位乘法：

    • 对于两个原码数，取其绝对值部分进行逐位相乘。
    • 乘法操作类似于普通的二进制乘法，但需要注意的是，对于每一位乘法，都需要考虑进位。

  3. 累加结果：

    • 逐位相乘的结果需要累加。此操作可以用类似于加法的方式进行。

  4. 处理最终结果的符号：

    • 根据两个操作数的符号，确定结果的符号。
      • 正数 × 正数：结果为正数。
      • 负数 × 负数：结果为正数。
      • 正数 × 负数 或 负数 × 正数：结果为负数。

  5. 调整结果的原码表示：

    • 将计算得到的结果转换回原码表示形式。
    • 如果结果是负数，需要将其绝对值的原码形式进行符号位设置。

  • -2 的原码： 1101

    • -2 的原码是 1101，符号为负。

  2. 取绝对值并转换：

    • 绝对值部分：3 → 0011，2 → 0010

  3. 逐位相乘：

    • 对 0011 和 0010 进行二进制乘法。
    • 计算步骤：

  4. 结果符号处理：

    • 由于原始乘数之一是负数，结果应为负数。

  5. 调整结果的原码表示：

    • 结果是 6 的绝对值部分为 0110。由于结果是负数，所以其原码形式为 1110。

4. 处理进位

在实际计算中，乘法涉及到进位的处理。每位乘法的进位需要在累加结果时考虑。例如，在逐位相乘过程中，若某位相乘产生进位，则需要在最终结果中加上这些进位。

5. 注意事项

  • 溢出问题：在进行乘法运算时，要注意可能出现的溢出问题。特别是在较高位数计算时，结果可能超出表示范围。
  • 计算复杂度：原码乘法涉及到逐位相乘和累加操作，这可能导致计算复杂度较高。在实际应用中，通常使用更高效的乘法算法（如补码乘法）。

6. 补码的乘法运算

补码的乘法运算在计算机中广泛使用，因为它能够简化符号位处理和加减运算。补码乘法与无符号数乘法类似，但需要特别处理符号位和补码转换。

1. 补码表示

  • 正数：补码与原码相同。例如，+5 的补码表示为 00000101（假设用 8 位表示）。
  • 负数：负数的补码是其原码的反码加1。例如，-5 的原码是 10000101，其反码为 11111010，因此补码为 11111011。

2. 乘法步骤

补码乘法的步骤可以分为以下几个部分：

  1. 取绝对值进行乘法运算：

    • 处理符号：根据乘数和被乘数的符号确定结果的符号。补码运算中，正数与负数的符号需要被处理好。
    • 将补码转为绝对值：对正数和负数，先将补码转为其绝对值的二进制数进行乘法运算。

  2. 逐位乘法：

    • 初始化：准备两个乘数的绝对值部分，初始化结果为零。
    • 逐位相乘：对每一位进行乘法操作，并考虑进位。

  3. 结果累加：

    • 将每一位乘法的结果加到总结果中。类似于传统的逐位乘法，但在计算过程中需要考虑进位。

  4. 调整结果符号：

    • 确定结果符号：根据操作数的符号，决定结果的符号。若操作数符号相同，则结果为正；若不同，则结果为负。
    • 将结果转换为补码：完成乘法后，结果需要转换回补码表示。

  • -2 的补码表示为 1110

    • -2 的补码是 1110，符号为负。

  2. 取绝对值并转换：

    • 绝对值部分：3 → 0011，2 → 0010

  3. 逐位相乘：

    • 对 0011 和 0010 进行二进制乘法。
    • 计算步骤：

    • 由于一个操作数是负数，所以结果应为负数。
    • 6 的补码表示（假设为 4 位）：
      • 正确的结果应为 -6 的补码。在 4 位表示下：
        • 6 的原码是 0110
        • 6 的反码是 1001
        • 6 的补码是 1010

  5. 最终结果：

4. 处理进位

在实际乘法运算中，进位的处理非常重要。在补码乘法中，进位是累加到结果中的。如果乘法结果超出了表示范围，需要对结果进行修正。

5. 溢出问题

在补码乘法中，溢出问题同样需要处理。特别是在有限位数的情况下，乘法结果可能会超出表示范围。溢出会影响最终的计算结果，通常需要根据位数进行处理，例如使用更多的位数来存储结果，或使用溢出检测机制来检查是否发生溢出。

7. 原码的除法运算

1. 恢复余数法

恢复余数法是一种逐步恢复余数并更新被除数的除法运算方法。具体步骤如下：

  1. 初始化：

    • 将被除数（Dividend）和除数（Divisor）设置为二进制数。
    • 初始化商（Quotient）为0，余数（Remainder）为被除数。

  2. 循环过程：

    • 取余数和被除数的最高位部分，与除数进行比较。
    • 如果余数部分大于或等于除数，则用除数从余数中减去，商的当前位设置为1。
    • 否则，商的当前位设置为0。
    • 将余数和被除数的下一位合并，更新余数。
    • 重复上述步骤，直到处理完所有位。

  3. 完成：

    • 最终商的所有位合并得到商。
    • 余数部分为最终的余数。

例子：假设我们要用恢复余数法计算 13 / 5（即1101 / 101）。

  1. 初始化：

    • 被除数：1101
    • 除数：101
    • 商：0000
    • 余数：1101

  2. 步骤：

    • 比较110（余数的前3位）与101（除数），110 ≥ 101。
      • 商的当前位设为1。
      • 余数更新为110 - 101 = 001。
      • 余数与下一个位合并，得到0011。
    • 比较001（余数的前3位）与101，001 < 101。
      • 商的当前位设为0。
      • 余数更新为001 - 0 = 0011。
      • 余数与下一个位合并，得到0110。
    • 比较011（余数的前3位）与101，011 < 101。
      • 商的当前位设为0。
      • 余数更新为011 - 0 = 0110。
      • 余数与下一个位合并，得到1100。
    • 比较110（余数的前3位）与101，110 ≥ 101。
      • 商的当前位设为1。
      • 余数更新为110 - 101 = 0011。

  3. 完成：

    • 商：0001（即1）
    • 余数：0011（即3）

2. 不恢复余数法

不恢复余数法是另一种除法运算方法，与恢复余数法不同的是，在整个除法过程中不需要恢复余数，直到最后一步才计算余数。具体步骤如下：

  1. 初始化：

    • 将被除数和除数设置为二进制数。
    • 初始化商为0。
    • 余数不进行初始化，直接计算。

  2. 循环过程：

    • 将被除数的高位部分与除数进行比较。
    • 如果高位部分大于或等于除数，则在商的当前位设为1，并用除数从被除数的当前高位部分中减去。
    • 否则，在商的当前位设为0。
    • 将被除数的下一位合并，更新被除数。
    • 重复上述步骤，直到处理完所有位。

  3. 完成：

    • 最终商为商的二进制数。
    • 余数为最后一次减法操作后的剩余部分。

例子：假设我们要用不恢复余数法计算 13 / 5（即1101 / 101）。

  1. 初始化：

    • 被除数：1101
    • 除数：101
    • 商：0000

  2. 步骤：

    • 比较110（前3位）与101，110 ≥ 101。
      • 商的当前位设为1。
      • 被除数更新为110 - 101 = 001。
      • 余数即为001。
      • 被除数与下一个位合并，得到0011。
    • 比较001（前3位）与101，001 < 101。
      • 商的当前位设为0。
      • 被除数更新为001 - 0 = 0011。
      • 被除数与下一个位合并，得到0110。
    • 比较011（前3位）与101，011 < 101。
      • 商的当前位设为0。
      • 被除数更新为011 - 0 = 0110。
      • 被除数与下一个位合并，得到1100。
    • 比较110（前3位）与101，110 ≥ 101。
      • 商的当前位设为1。
      • 被除数更新为110 - 101 = 0011。

  3. 完成：

    • 商：0001（即1）
    • 余数：0011（即3）
  • 恢复余数法：每一步都恢复余数，并根据恢复余数更新被除数，直至完成整个除法过程。
  • 不恢复余数法：仅在除法完成后计算最终余数，通过逐步更新被除数和商。

8. 补码的除法

补码（Two’s Complement）是一种用于表示有符号整数的方法。进行补码除法时，我们需要考虑负数的结果和补码表示。以下是补码除法的关键点：

  1. 除法过程：

    • 商的符号：商的符号由被除数和除数的符号决定。如果两者符号相同，则商为正；如果符号不同，则商为负。
    • 余数的符号：余数的符号与被除数的符号相同。

  2. 步骤：

    • 确定符号：计算商和余数的符号。
    • 计算绝对值：将被除数和除数的绝对值进行除法运算。
    • 转换为补码：将计算结果转换为补码形式。

  3. 补码除法例子：

    • 对于8位二进制补码，假设被除数为 -14，除数为 3，我们可以将 -14 表示为补码，然后进行除法计算，最终得到商和余数。

10. 数据的存储和排列

数据在计算机中以特定格式存储，不同类型数据的存储方式包括：

  1. 定点数和浮点数存储：

    • 定点数：以固定的小数位数存储，常用于表示精确数值，如货币。
    • 浮点数：以科学计数法的形式存储，适用于表示很大或很小的数值，如 float 和 double 类型。浮点数包括一个符号位、一个指数部分和一个尾数部分。

  2. 字节序（Endianess）：

    • 大端序（Big-endian）：数据的高位字节存储在低地址，低位字节存储在高地址。
    • 小端序（Little-endian）：数据的低位字节存储在低地址，高位字节存储在高地址。

  3. 示例：

    • 对于32位整数 0x12345678，在大端序系统中，它会被存储为 12 34 56 78；在小端序系统中，它会被存储为 78 56 34 12。

11. 浮点数的表示

1. 规格化（Normalization）

规格化是将浮点数表示为科学计数法的标准形式。规格化的目的是将浮点数的尾数（Mantissa 或 Fraction）放在一个特定的范围内，以便于计算和表示。

  • 科学计数法表示：浮点数的表示形式通常是 ($ M \times 10^E $)，其中 ( M ) 是尾数（或称为有效位），( E ) 是指数。

  • 规格化要求：在二进制浮点数中，尾数通常被规范为在某个范围内（例如，对于二进制浮点数，尾数在 ( [1, 2) ) 的范围内）。这使得数值表示更为精确，利用尾数的有效位提高计算精度。

2. 规格化例子

  • 数值 ( 0.15625 )：
    • 转换为二进制：0.00101。
    • 规格化：$1.01 \times 2^{-3}$。

12. IEEE 754 浮点数标准

IEEE 754 是浮点数表示的标准，定义了浮点数在计算机中的表示方式。它包括单精度（32位）和双精度（64位）格式。

1. 单精度浮点数（32位）

  • 结构：32位浮点数由以下三部分组成：

    • 符号位（1位）：表示数值的符号，0 为正数，1 为负数。
    • 指数位（8位）：表示指数的值，用于调整尾数的位置。IEEE 754 使用偏移量（Bias）方法，单精度的偏移量为 127。
    • 尾数位（23位）：表示规格化后的尾数部分（隐含的 1 不被存储）。

    格式示意：

    • S：符号位
    • E：指数位
    • M：尾数位

  • 示例：

    • 数值 ( 5.75 ) 的单精度表示：
      • 二进制表示：101.11
      • 规格化：$1.0111 \times 2^2$
      • 符号位：0（正数）
      • 指数：2 + 127 = 129，二进制为 10000001
      • 尾数：01110000000000000000000
      • 最终表示：0 10000001 01110000000000000000000

2. 双精度浮点数（64位）

  • 结构：64位浮点数由以下三部分组成：

    • 符号位（1位）：表示数值的符号。
    • 指数位（11位）：表示指数的值，偏移量为 1023。
    • 尾数位（52位）：表示规格化后的尾数部分（隐含的 1 不被存储）。

    格式示意：

    • S：符号位
    • E：指数位
    • M：尾数位

  • 示例：

    • 数值 ( 5.75 ) 的双精度表示：
      • 二进制表示：101.11
      • 规格化：$1.0111 \times 2^2$
      • 符号位：0（正数）
      • 指数：2 + 1023 = 1025，二进制为 10000000001
      • 尾数：0111000000000000000000000000000000000000000000000000
      • 最终表示：0 10000000001 0111000000000000000000000000000000000000000000000000

13. 浮点数的运算

浮点数的运算包括加法、减法、乘法和除法。为了进行这些运算，必须处理以下几个步骤：对阶、尾数加减、规格化、舍入以及判溢出。

（1）对阶（Alignment）

对阶是将两个浮点数调整为相同的指数，以便进行加法或减法运算。

  • 步骤：

    1. 比较指数：将两个浮点数的指数部分进行比较，确定哪个指数较大。
    2. 调整尾数：将指数较小的浮点数的尾数右移（相当于除以 2）直到两个浮点数的指数相同。每移动一位，相应的尾数增加一个零。
    3. 结果：两个浮点数现在具有相同的指数，可以进行加法或减法运算。

  • 示例：

    • 假设有两个浮点数 ( $A = 1.25 \times 2^3$ ) 和 ( $B = 0.5 \times 2^2$ )。
    • 将 ( B ) 的尾数右移1位，使其变为 ($ 0.25 \times 2^3 $)。
    • 现在 ( $A = 1.25 \times 2^3 $) 和 ( $B = 0.25 \times 2^3 $)，可以进行加法或减法运算。

（2）尾数加减

尾数加减是在对阶之后进行的实际加法或减法操作。

  • 步骤：

    1. 尾数加法：将对阶后的两个尾数进行加法操作。
    2. 尾数减法：将对阶后的两个尾数进行减法操作。

  • 示例：

    • 假设 ( $A = 1.25 \times 2^3$ ) 和 ($ B = 0.5 \times 2^3 $)。
    • 尾数相加：( 1.25 + 0.5 = 1.75 )。
    • 尾数相减：( 1.25 - 0.5 = 0.75 )。

（3）规格化

规格化是将运算结果调整为科学计数法的标准形式，以保持尾数在规定的范围内（通常是 ( [1, 2) )）。

  • 步骤：

    1. 检查尾数：如果尾数的绝对值大于等于 2，或者小于 1，则需要规格化。
    2. 调整尾数：将尾数调整到 ( [1, 2) ) 的范围内，同时调整指数。
    3. 结果：尾数在标准范围内，指数经过相应的调整。

  • 示例：

    • 计算结果为 ($ 2.75 \times 2^3$ )，需要规格化为 ($1.375 \times 2^4 $)。

（4）舍入

浮点运算结果通常需要舍入以适应浮点数的有限精度。常用的舍入方式包括：

  • 向上舍入（Ceiling Rounding）：将结果舍入到最接近的上限值。
  • 向下舍入（Floor Rounding）：将结果舍入到最接近的下限值。

  • 四舍五入（Round Half-Up）：将结果舍入到最接近的整数，四舍五入到最接近的数值。

  • 示例：

    • 若结果为 ( 1.987 ) 并且需要四舍五入到两位小数，结果为 ( 1.99 )。
    • 若使用向下舍入，结果为 ( 1.98 )。

（5）判溢出

浮点数运算中需要检测是否发生溢出，即结果是否超出可表示的范围。

  • 步骤：

    1. 计算范围：确定浮点数的最大值和最小值。
    2. 检查溢出：在运算后检查结果是否超出了表示范围。
    3. 处理溢出：如果发生溢出，结果通常被设为无穷大（positive infinity）或负无穷大（negative infinity），或者产生溢出异常。

  • 示例：

    • 对于32位单精度浮点数，其最大值为 ( $3.4028235 \times 10^{38} $)。如果运算结果大于这个值，则发生溢出，结果可能被设为 ($+\infty$ )。

14. 电路的基本原理、加法器设计

加法器是数字电路中的基本组件，用于执行二进制加法运算。基本的加法器包括半加器和全加器，而设计中的加法器可以分为串行加法器和并行加法器。

1. 基本原理

半加器（Half Adder）

半加器是最简单的加法器，用于计算两个单比特的和和进位。其功能是计算两个比特的和及产生的进位。

  • 功能：给定两个输入比特 ( A ) 和 ( B )，半加器输出两个信号：

    • 和（Sum, S）：计算 ( A ) 和 ( B ) 的异或（XOR）运算。
    • 进位（Carry, C）：计算 ( A ) 和 ( B ) 的与（AND）运算。

  • 逻辑电路：

    • 和（Sum）：( $S = A \oplus B $)
    • 进位（Carry）：( $C = A \land B$)

全加器（Full Adder）

全加器用于计算三个比特的和：两个加数和一个来自前一位的进位。它可以看作是由两个半加器和一个或门（OR Gate）组合而成的。

  • 功能：给定三个输入比特 ( A )、( B ) 和进位输入 ( C_{in} )，全加器输出两个信号：

    • 和（Sum, S）：计算 ( A ) 和 ( B ) 以及进位输入的异或（XOR）运算。
    • 进位输出（$Carry-out, C_{out}$）：计算产生的进位。

  • 逻辑电路：

    • 和（Sum）：($ S = A \oplus B \oplus C_{in} $)
    • 进位输出（Carry-out）：( $C_{out} = (A \land B) \lor (C_{in} \land (A \oplus B))$ )

2. 加法器设计

串行加法器（Serial Adder）

串行加法器逐位处理两个二进制数，加法是按位依次进行的，从最低位开始处理，依次计算进位，并在下一位使用。

  • 工作原理：

    1. 从最低有效位（LSB）开始加法，计算和和进位。
    2. 将进位传递到下一位，重复以上过程。
    3. 处理的速度较慢，但电路较简单。

  • 电路组成：

    • 多个全加器级联，每个全加器处理当前位和来自前一位的进位。

并行加法器（Parallel Adder）

并行加法器一次性处理所有位的加法。它使用多个全加器并行计算，具有更快的加法速度。

  • 进位前加器（Carry-Lookahead Adder, CLA）：

    • 工作原理：通过预先计算每个位的进位，减少进位传递时间。
    • 逻辑：
      • 生成信号（Generate, G）：($ G_i = A_i \land B_i $)
      • 传播信号（Propagate, P）：($ P_i = A_i \oplus B_i $)
      • 进位计算：

    • 逻辑：将加法分成多个部分（块），每个块计算两种进位情况（0 和 1），并通过选择电路最终得到结果。

  • 串行加法器 是逐位处理的加法器，简单但速度较慢。
  • 并行加法器 提供更快的加法，通过 进位前加器 和 进位选择加法器 解决进位传递的问题。进位前加器使用预计算的方法提高速度，而进位选择加法器则通过分块处理和选择电路来实现更快的加法。

15. 加法器、ALU的改进

加法器和算术逻辑单元（ALU）是数字电路和计算机体系结构中的关键组件。对这些组件的改进旨在提高计算速度和性能。

加法器改进

1. 超前进位加法器（Carry-Skip Adder, CSA）

超前进位加法器是一种改进的并行加法器，通过跳过部分进位计算来提高加法速度。

  • 工作原理：

    • 分组：将加法操作分为多个块，每个块的进位计算通过跳过操作来减少延迟。
    • 跳过：如果某个块的进位生成信号为真（即该块产生进位），则可以跳过块内的其他进位计算，而直接使用块外的进位信号。

2. 动态加法器（Dynamic Adder）

动态加法器利用动态电路技术提高速度和功耗效率。

  • 工作原理：

    • 动态电路：使用动态逻辑（如电荷泵电路）来减少逻辑门的延迟。
    • 预充电：在计算之前，通过预充电电路将电路状态准备好，从而提高加法器的速度。

  • 特点：

    • 速度：动态加法器在高速计算中表现优越。
    • 功耗：动态电路可以通过优化功耗来提升性能，但设计复杂度较高。

3. 超前进位加法器（Carry-Lookahead Adder, CLA）

虽然前面提到的 CLA 是一种并行加法器的类型，但它也可以被改进以处理更高位数的进位问题。

  • 工作原理：

    • 生成与传播：提前计算每个比特位的进位生成（Generate）和传播（Propagate）信号。
    • 并行计算：通过逻辑门并行计算进位，显著减少延迟。

  • 改进：

    • 层次化设计：将 CLA 设计分层，减少每层的进位计算时间，提高整体计算速度。

ALU改进

1. 多功能ALU

现代ALU通常支持多种算术和逻辑操作。为了提高性能和灵活性，ALU 的设计可以包括以下改进：

  • 功能扩展：

    • 新指令：增加对更多数学运算、位操作和数据处理操作的支持。
    • 复合操作：支持同时进行多个操作，如加法和位移操作的复合。

  • 执行单元：

    • 流水线：使用流水线技术将多个操作阶段分开，提高运算吞吐量。
    • 并行处理：增加多个执行单元以并行处理不同类型的操作。

2. 设计优化

  • 优化时钟频率：

    • 时钟优化：通过优化时钟周期和减少时钟延迟，提高 ALU 的工作频率。
    • 异步设计：使用异步电路技术减少时钟延迟，提高性能。

  • 功耗管理：

    • 动态功耗控制：根据工作负载动态调整电源供应，降低功耗。
    • 静态功耗减少：通过改进电路设计减少静态功耗，如使用低功耗逻辑门。

  • 高速缓存和寄存器：

    • 寄存器组：增加寄存器组以减少对主存储器的访问需求，提高处理速度。
    • 缓存优化：使用高速缓存来存储频繁访问的数据，减少延迟。

3. 高级ALU架构

  • 浮点运算单元：

    • 专用单元：为浮点运算设计专用硬件单元，提高浮点运算的速度和精度。
    • IEEE 754标准支持：确保浮点运算符合 IEEE 754 标准，包括浮点加法、乘法、除法和其他操作。

  • 矢量处理单元：

    • SIMD（Single Instruction, Multiple Data）：支持同时对多个数据进行相同操作，提高数据处理效率。
    • 向量寄存器：使用向量寄存器处理大量数据，提高计算密度和效率。
