计算机组成原理之计算机系统概述

1. 计算机发展历程

1.1 计算机的硬件发展

  1. 电子管时代 (1940s-1950s)：

    • 计算机的初始时代，以电子管为主要元件。
    • 电子管体积大、功耗高、发热严重，计算机体积庞大，故障率高。
    • 代表：ENIAC（世界上第一台电子数字计算机）。

  2. 晶体管时代 (1950s-1960s)：

    • 晶体管的发明大大减小了计算机的体积，降低了功耗和发热量。
    • 计算机性能和稳定性显著提高。
    • 代表：第二代计算机，如IBM 7090。

  3. 中小规模集成电路时代 (1960s-1970s)：

    • 中小规模集成电路的出现使得成千上万的电子元件集成在一个小芯片中。
    • 计算机性能和效率进一步提升，体积缩小，成本降低。
    • 代表：第三代计算机，如IBM 360。

  4. 超大规模集成电路时代 (1970s-至今)：

    • 超大规模集成电路 (VLSI) 的发展使得数百万个晶体管可以集成到一个芯片上，形成了微处理器。
    • 计算机开始进入个人计算机时代，微型化、低成本化，广泛进入大众市场。
    • 代表：第四代计算机，如Intel 8086。

元件更新变化：

  1. 摩尔定律：

    • 摩尔定律指出，在每18个月，芯片上的晶体管数量会翻一倍，从而提高计算机的性能。

  2. 半导体存储器的不断发展：

    • 早期的计算机使用磁鼓、磁芯等作为存储介质，后来发展为半导体存储器，速度更快、容量更大。

  3. 微处理器的不断发展：

    • 微处理器是计算机中央处理器 (CPU) 的核心，随着晶体管技术的发展，微处理器的性能和集成度不断提高，推动了计算机的高速化和小型化。

1.2 计算机软件发展

  1. 面向机器的语言：

    • 机器语言：计算机最底层的语言，由二进制代码直接控制硬件操作，编写复杂且易出错。
    • 汇编语言：使用符号指令代替二进制指令，使程序编写相对简单，但仍然依赖具体的硬件体系结构。

  2. 面向问题的高级语言：

    • 高级语言抽象了硬件细节，更关注问题的解决方案。通过编译器或解释器将代码翻译为机器指令。
    • 代表语言：C、C++、Java 等。高级语言更易于理解、移植性强，极大提高了编程效率。

1.3 计算机的分类与发展方向

  1. 电子计算机：

    • 电子模拟计算机：通过模拟物理系统的方程来进行计算，适用于模拟连续变化的系统。
    • 电子数字计算机：处理离散的数值和逻辑运算，是现代计算机的主要形式。

  2. 数字计算机的分类：

    • 专用计算机：设计用于特定任务，如工业控制、科学计算等。
    • 通用计算机：能够执行多种任务，根据规模和功能进一步分为：
      • 巨型机：用于科学计算、气象预报、核试验等超级计算任务。
      • 大型机：用于企业级应用和海量数据处理。
      • 中型机、小型机、微型机：逐渐面向中小企业和个人用户。
      • 单片机：集成CPU、内存、输入输出设备于一体，应用于嵌入式系统。

  3. 按指令和数据流分类：

    • SISD（单指令流单数据流）：冯·诺依曼体系结构，传统的串行计算模型。
    • SIMD（单指令流多数据流）：用于向量处理和阵列处理器系统，如GPU。
    • MISD（多指令流单数据流）：理论上存在，但实际应用较少。
    • MIMD（多指令流多数据流）：多处理器和多计算机系统，如现代多核处理器。

计算机发展趋势：

  1. 微型计算机：

    • 向更微型化、网络化、高性能和多用途方向发展，未来将进一步渗透至个人生活和物联网领域。

  2. 巨型计算机：

    • 走向更巨型化、超高速化、并行处理和智能化。随着人工智能、云计算和大数据的发展，巨型计算机将成为解决复杂问题的重要工具。

2. 计算机系统结构

2.1 计算机系统构成

计算机系统由硬件系统和软件系统两大部分组成：

  • 硬件系统：指计算机的物理设备，如处理器、存储器、输入输出设备等。
  • 软件系统：指控制和管理硬件工作、以及解决特定问题的程序系统，包括系统软件和应用软件。

2.2 计算机硬件的基本组成

2.2.1 冯·诺依曼机

冯·诺依曼计算机体系结构由以下五个主要部件组成：

  1. 存储器：用于存储数据和程序。
  2. 运算器：执行算术运算和逻辑运算，并保存中间结果。
  3. 控制器：控制指令的执行，协调各部件之间的操作。
  4. 输入设备：将外部数据输入计算机，如键盘、鼠标等。
  5. 输出设备：将计算结果以人类能理解的形式输出，如显示器、打印机等。

冯·诺依曼机的特征：

  • 使用二进制数制来表示数据和指令。
  • 程序和数据存储在同一个存储器中，计算机按照程序的指令顺序执行。
  • 以运算器为核心，各部件协调工作。

2.2.2 现代计算机的组织结构

现代计算机的发展更加以存储器为中心，使得I/O 操作尽可能绕过 CPU，直接在 I/O 设备与存储器之间进行数据传输。这样可以大大提高系统的整体运行效率，减轻 CPU 的负担，优化系统性能。

  • DMA（Direct Memory Access，直接存储器访问）技术使 I/O 设备能直接与存储器交换数据，不需要 CPU 的干预。

2.3 计算机软件的分类

计算机软件按功能可以分为两大类：

  1. 系统软件：

    • 这些软件负责管理计算机资源，确保计算机高效、正确运行。
    • 包括操作系统（如 Windows、Linux）、数据库管理系统等。

  2. 应用软件：

    • 用户为了特定应用领域编写的程序，用来解决具体问题。
    • 如办公软件、科学计算软件、游戏软件等。

按语言层次分类：

  1. 机器语言：

    • 机器语言是计算机唯一能够直接识别和执行的语言，使用二进制编码，最为底层。

  2. 汇编语言：

    • 汇编语言使用助记符代替机器指令，需要经过汇编器转化为机器语言才能执行。

  3. 高级语言：

    • 高级语言如 C、C++、Java 使用更接近人类语言的符号，经过编译器编译或解释器解释后可执行。

2.4 计算机工作过程

计算机执行程序的基本过程如下：

  1. 程序和数据装入主存：程序和数据从外存（如硬盘）加载到主存（RAM）。

  2. 从程序的起始地址开始运行：程序计数器（PC）初始化，指向程序的首地址。

  3. 取指令：从存储器中读取一条指令到指令寄存器（IR）。

  4. 分析指令：控制单元（CU）分析指令的操作码（OP），并生成相应的控制信号。

  5. 执行指令：执行指令指定的操作，并根据结果更新寄存器或存储器的状态。

  6. 重复步骤：依次读取、分析、执行每条指令，直到程序结束。

信息流程：

  • 取指令：PC -> MAR -> M（存储器）-> MDR -> IR
  • 分析指令：OP（IR） -> CU
  • 执行指令：Ad（IR） -> MAR -> M -> MDR -> ACC

2.5 多级层次结构

计算机系统的硬件和软件设计通常采用多级层次结构，将复杂的系统分解成多个层次，每一层次处理特定的功能，从而简化系统设计和优化性能。例如：

  • 硬件层：物理设备和电路层。
  • 微程序层：底层的控制逻辑。
  • 操作系统层：系统资源的管理。
  • 应用程序层：用户使用的应用程序。

2.6 各硬件部件

  1. 主存（内存）：

    • 存储器是存放数据和程序的地方，通常包括以下概念：
      • 存储元：存储最小的二进制信息单元。
      • 存储单元：存储一个完整的数据项（如一个字节）。
      • 存储字：存储单元的集合，用于存储处理器能一次处理的位数。
      • 存储字长：每个存储字的位数。
      • MAR（Memory Address Register，地址寄存器）：存储器的地址寄存器，保存当前要访问的内存地址。
      • MDR（Memory Data Register，数据寄存器）：保存从存储器读取或写入的数据。

  2. 运算器：

    • ACC（累加寄存器）：用于暂存运算结果。
    • MQ（乘商寄存器）：用于存储乘法和除法的中间结果。
    • X（通用寄存器）：用于存储临时数据。
    • ALU（算术逻辑单元）：执行算术和逻辑运算。

  3. 控制器：

    • PC（程序计数器）：指向下一条将要执行的指令的地址。
    • IR（指令寄存器）：保存当前正在执行的指令。
    • CU（控制单元）：生成控制信号，指挥整个计算机系统的工作。

计算机的工作过程详细分析

当程序被加载到主存储器中时，程序计数器（PC）指向程序的第一条指令。执行过程如下：

  1. 取指令：程序计数器 (PC) 的地址加载到地址寄存器 (MAR) 中，存储器 (M) 将该地址的内容加载到数据寄存器 (MDR) 中，随后指令被传送到指令寄存器 (IR) 中。
  2. 分析指令：控制单元 (CU) 对指令进行译码，确定操作类型和操作数。
  3. 执行指令：根据指令操作数的地址，访问存储器或寄存器，将数据传送到累加寄存器 (ACC) 或其他寄存器中，最终进行算术或逻辑运算，或者执行 I/O 操作。

3 计算机性能指标

计算机的性能可以通过多个指标来衡量，它们涵盖从处理数据的速度到存储器的容量，以下是一些主要的计算机性能指标：

3.1 机器字长

机器字长是指计算机在一次整数运算（即定点整数运算）中所能处理的二进制数据的位数。它与 CPU 的内部寄存器的位数和加法器有关。机器字长通常等于 CPU 内部寄存器的大小。字长越大，计算机能够处理的数据范围越广，计算精度越高。常见的字长有 16 位、32 位、64 位等，字长通常为 8 位（即一个字节）的整数倍。字长决定了 CPU 每次可以处理的数据的宽度，也影响数据在系统内的传递和处理速度。

3.2 数据通路带宽

数据通路带宽指的是计算机系统中的数据总线一次能够并行传输的数据位数。数据总线的宽度通常和计算机的字长相关，影响系统的数据传输能力。带宽越宽，计算机每次能够传输的数据量就越大，因此带宽决定了系统的数据处理速度。常见的数据通路带宽有 32 位、64 位等，现代高性能计算机通常使用 64 位或更高的带宽。

3.3 主存容量

主存储器容量是指计算机主存储器可以存储信息的最大容量，通常以字节（Byte）为单位来衡量。主存容量通常通过 “字数 * 字长” 来表示，即主存储器中的存储单元数量乘以每个单元能够存储的数据位数。主存容量越大，计算机可以处理的程序和数据集越大。常见的主存容量为 8GB、16GB、32GB 或更高。

3.4 运算速度

3.4.1 吞吐量和响应时间

3.4.1.1 吞吐量
吞吐量（Throughput）是指计算机系统在单位时间内处理请求的数量。吞吐量直接影响系统的处理能力，取决于输入数据能够多快地被系统接收、处理和存储。对于高负载的系统，如服务器或数据中心，吞吐量是一个非常重要的性能指标。

3.4.1.2 响应时间
响应时间（Response Time）是指用户向计算机发送请求，到系统对该请求做出响应并返回结果之间的时间间隔。响应时间反映了系统的交互性能，对于实时系统或交互密集的应用，响应时间越短，用户体验越好。

3.4.2 主频和 CPU 时钟周期

3.4.2.1 CPU 时钟周期
CPU 时钟周期（Clock Cycle）是指 CPU 完成基本操作（如取指令、执行指令等）所需的最小时间单位。一个时钟周期的时长是主频的倒数，因此时钟周期越短，CPU 每秒可以完成的基本操作就越多。

3.4.2.2 主频
主频（Clock Frequency）也称为时钟频率，单位是赫兹（Hz）。主频表示 CPU 的运行速度，通常用 GHz（十亿赫兹）来表示。例如，1.8GHz 的 CPU 每秒钟可以执行 18 亿个时钟周期。主频越高，CPU 执行指令的速度就越快。但主频并不是唯一决定性能的因素，还要考虑 CPU 结构、缓存、指令集等。

3.4.2.3 CPI
CPI（Cycles Per Instruction）表示每条指令执行所需的时钟周期数。不同指令的 CPI 不同，CPI 越低，CPU 执行指令的效率越高。实际 CPU 性能由主频和 CPI 共同决定。

3.4.2.4 CPU 执行时间
CPU 执行时间是指运行一个程序所花费的总时间，可以通过以下公式计算：
$\text{CPU 执行时间} = \frac{\text{CPU 时钟周期数}}{\text{主频}} = \frac{\text{指令条数} \times \text{CPI}}{\text{主频}}$

这个公式说明了 CPU 执行时间取决于执行的指令数量、每条指令的 CPI 以及主频。

3.4.3 MIPS、MFLOPS、GFLOPS、TFLOPS

MIPS（Million Instructions Per Second）表示每秒能够执行的百万级指令条数。MIPS 是衡量计算机整数运算能力的常用指标，通常用于描述 CPU 的性能。尽管 MIPS 是一个常见的指标，但它并不能完全反映 CPU 的实际性能，因为不同的指令执行时间不同。

MFLOPS（Million Floating-Point Operations Per Second）表示每秒能够执行的百万次浮点运算。MFLOPS 是衡量计算机浮点运算性能的重要指标，特别适用于科学计算和图形处理领域。

GFLOPS（Giga Floating-Point Operations Per Second）表示每秒能够执行的十亿次浮点运算，通常用于描述 GPU 和高性能计算系统的性能。GFLOPS 指标越高，系统处理复杂计算任务的能力越强。

TFLOPS（Tera Floating-Point Operations Per Second）表示每秒能够执行的万亿次浮点运算。TFLOPS 通常用于衡量超级计算机或顶级 GPU 的性能，特别是在需要极高浮点运算能力的领域，如气象模拟、大数据分析和 AI 训练等。TFLOPS 越高，表示该系统处理复杂计算任务的能力越强。然而，在实际应用中，TFLOPS 并不能完全反映 GPU 或计算机的实际表现，还要考虑任务的并行化、内存带宽、I/O 性能等。
