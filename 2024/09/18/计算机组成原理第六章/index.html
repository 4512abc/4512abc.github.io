计算机组成原理之总线

总线概述

总线基本概念

总线: 总线是计算机中连接各个部件的信息传输线，是各个部件共享的传输介质，起到信号的公共传输线的作用。

  • 功能:

    • 作为各功能部件之间传递信息的通道。
    • 作为部件间的物理接口，减少通信复杂度。
    • 提供数据、地址、时序和控制信息的交换。
    • 规定共同遵循的协议或标准。
    • 避免成为计算机性能的瓶颈。
    • 方便计算机系统的集成、扩展和升级。

  • 信息传送方式:

    • 串行: 一位一位地传送信息，适合较长距离的传输。
    • 并行: 多位同时传送信息，适合较短距离的传输，但容易产生干扰和信号变形。

总线分类

  1. 根据位置:

    • 片内总线（内部总线）: 芯片内部的总线，用于连接CPU内部的寄存器和运算器部件。
    • 系统总线（外部总线）: 用于连接计算机各种部件，包括：
      • 数据总线（DB）: 传输数据信号，双向，与机器字长、存储字长有关。
      • 地址总线（AB）: 传输地址信号，单向，与存储地址、I/O地址相关。
      • 控制总线（CB）: 传输控制信号和状态信号，包括读/写、总线请求等。

  2. 通信总线（I/O总线）:

    • 串行通信总线: 支持较长距离的通信，包括同步和异步方式。
    • 并行通信总线: 支持较短距离的通信。

总线特性及性能指标

  1. 总线特性:

    • 机械特性: 尺寸、形状、管脚数、排列顺序等物理连接方式。
    • 电气特性: 传输方向和有效电平范围。
    • 功能特性: 每根线的功能，如地址、数据、控制、状态反馈信号。
    • 时间特性: 信号的时序关系。

  2. 性能指标:

    • 总线带宽: 总线的最高传输速率，单位为MB/s，与数据线的数量有关。
    • 总线频率（f）: 反映总线工作速率，单位为MHz。
    • 总线宽度（w）: 数据总线的位数，通常与处理器的字长一致。
    • 总线传输速率（BW）: 计算公式为 ( $\text{BW} = \left(\frac{\text{总线宽度}}{8}\right) \times \text{总线频率}$ )，单位为MB/s。
    • 标准传输率: 每秒传输的最大字节数。
    • 时钟同步/异步: 表示总线是否需要同步时钟。
    • 总线复用: 地址线与数据线复用的情况。
    • 信号线数: 包括地址线、数据线、控制线的总和。
    • 总线控制方式: 包括突发、自动、仲裁、逻辑、计数等。
    • 其他指标: 负载能力等。

总线结构

  1. 单总线结构:

    • 使用一条系统总线连接CPU、主存和I/O设备。
    • 总线简单，易于扩展，但可能成为系统瓶颈，适用于慢速计算机系统。

  2. 双总线结构:

    • 面向CPU的双总线结构: 分开CPU与主存、I/O设备的总线。
    • 以存储器为中心的双总线结构: 存储器与I/O设备通过双总线结构连接。
    • 主存-I/O双总线结构: 通过通道统一管理I/O。

  3. 多总线结构:

    • CPU与cache之间使用高速总线，主存连接在系统总线上，还设有桥和扩展总线接口。

    • 这种结构下，速度差异较大的设备使用不同速度的总线。

    • 总线桥: 连接不同速率总线的设备，起到信号缓冲、电平转换、控制协议转换等作用。

  4. 单机总线与多级总线的区分标志:

    • 不仅仅看总线数量，若指令中有单独的I/O总线，则为双级或多级总线结构，否则为单机总线。

总线结构对计算机系统性能的影响:

  • 对最大存储容量的影响: 单总线结构有影响，双总线和多总线结构没有。
  • 对指令系统的影响: 双总线和多总线结构需要增加I/O指令。
  • 吞吐量: 单总线结构吞吐量小，多总线结构吞吐量大，双总线结构居中。

总线接口（I/O接口）概述

概念: 也称为I/O接口或I/O设备控制器，是CPU与主存、外围设备之间通过总线连接的逻辑部件。

作用:

  • 将多种类的外设连接到总线中。
  • 处理外设速度较慢与存储器速度较快之间的差异。
  • 转换外设信号为处理器可以识别的信号。

基本结构:

  • 端口: CPU直接访问的I/O接口内部寄存器。
  • 端口号: 每个端口的访问地址。

I/O接口的功能:

  • 控制功能: 接收CPU控制命令，启动或关闭设备。
  • 缓冲功能: 作为缓冲器补偿速度差异。
  • 状态功能: 监视和保存外围设备的工作状态。
  • 转化功能: 完成数据转换，如串行-并行和并行-串行转换。
  • 程序中断功能: 当外设需要CPU操作时发出中断请求。
  • 复位功能: 接收复位信号重启接口和外设。
  • 错误检查功能: 校验输入输出数据，检查传送错误。

总线通信

相关概念

波特率 (Baud Rate):

  • 波特率是描述串行通信系统每秒传输的信号变化次数的单位。每次变化可能代表一个比特位，因此波特率通常与数据传输速率相关。例如，波特率为9600的通信系统每秒钟可以传送9600个比特位的数据。

并行处理:

  • 在计算机系统中，并行处理指同时处理多个数据位。为了进行并行通信，数据被分成多个部分，每部分通过不同的信号线同时传送。与串行通信相比，并行处理能够更快地传输大量数据，因为它能在相同时间内传送更多信息。

并到串变换（拆卸）和串到并变换（装配）:

  • 并到串变换: 在发送端，将多个数据线上的并行数据转换成串行数据。这涉及到将多个数据位按照顺序合并成一个数据流进行传送。
  • 串到并变换: 在接收端，将串行数据流拆分成多个并行数据位，并分别传输到各自的数据线。这是将接收到的串行数据重新组装成并行形式的过程。

信息传递方式

  1. 串行通信:

    • 定义: 在串行通信中，数据位一个接一个地按顺序通过单一信道传送。每个比特位在固定的时间间隔内发送，这种传输通常是通过脉冲信号实现的。在异步串行通信中，数据的开始和结束由特定的起始和停止信号标记，而在同步串行通信中，数据传输由同步时钟信号进行协调。

    • 特点:

      • 速度: 比并行通信慢，因为数据逐位传送。但在某些情况下，随着技术的进步，串行通信的速度也可以非常高。
      • 成本: 低，因为只需要一条数据线进行传输，适合长距离传输。
      • 干扰: 较小，因为只使用一条线，减少了信号干扰的可能性。
      • 适用场景: 远程通信，诸如串口通信、网络通信等。

    • 优点:

      • 简化布线: 减少了传输线的数量，降低了布线成本和复杂性。
      • 适应性强: 对长距离传输具有更好的适应性，信号衰减较小。

    • 缺点:

      • 传输速率: 相较于并行通信，传输速率较低。

  2. 并行通信:

    • 定义: 在并行通信中，数据的多个比特位通过多条并行的传输线同时传送。每条线负责传输数据的一个比特位。通常，通信协议会使用应答信号来协调数据的传输，以确保数据完整性和同步。

    • 特点:

      • 速度: 实际传输速度快，因为多个数据位同时传输。适合需要高数据传输速率的场景。
      • 成本: 高，尤其在长距离传输时，电缆的成本和复杂度增加。
      • 干扰: 由于多条数据线之间的信号可能干扰，可能导致信号的失真。
      • 适用场景: 计算机内部组件的通信，如CPU与内存之间的数据传输。

    • 优点:

      • 高速度: 能够在短时间内传输更多的数据。
      • 数据完整性: 通常支持更高的数据完整性检查。

    • 缺点:

      • 成本高: 多条传输线增加了布线成本。
      • 干扰问题: 长距离传输时，线间干扰会影响信号质量。

  3. 串并传送:

    • 定义: 在串并传送中，虽然一次传输的数据宽度小于整个数据宽度，但数据以串行方式分多次传输，从而使得整个数据宽度得以传输。

    • 特点:

      • 效率: 通过多次串行传输完成数据传送，兼顾了串行和并行通信的优点。
      • 适用场景: 适用于需要在速度和成本之间找到平衡的应用场景。

    • 优点:

      • 平衡性能与成本: 能在较长距离内保持较高的传输速率，同时控制成本。
      • 灵活性: 结合了串行和并行传输的优点，提高了系统的灵活性。

    • 缺点:

      • 复杂性: 实现和管理较为复杂，需要在硬件和协议上进行调整。
  • 串行通信 适合远距离、成本较低的传输需求，但速度较慢。
  • 并行通信 适合短距离、高速传输需求，但成本高且可能受干扰。
  • 串并传送 在速度和成本之间提供了一个折中方案，适合中等距离和需要高效传输的场景。

总线控制

总线事务

总线事务是指从一个设备请求总线到完成使用总线的整个操作序列。一个完整的总线事务包括以下几个主要步骤：

  1. 请求操作: 主设备向总线发出请求，说明它需要使用总线进行数据传输。
  2. 裁决操作: 总线仲裁器决定是否授予主设备总线使用权。这个步骤可能涉及多个设备竞争总线的情况。
  3. 地址传输: 主设备在获得总线控制权后，发送目标设备的地址以及相关的控制命令。
  4. 数据传输: 主设备和从设备之间进行实际的数据交换。
  5. 总线释放: 数据传输完成后，主设备释放总线，使其他设备可以使用。

总线判优控制

总线的判优控制确保在同一时刻只有一个主设备可以控制总线。这主要涉及两个方面：

  1. 设备分类:

    • 主设备: 可以提出总线占用申请，并在获得控制权后进行数据传输。通常，CPU或者其他高优先级设备被设计为主设备。
    • 从设备: 不能主动请求总线，只能响应主设备的请求。它们通常只执行接收或发送数据的操作。
    • 有些设备既可以作为主设备也可以作为从设备，这取决于具体的应用场景和设计。

  2. 总线仲裁:

    • 集中式仲裁: 所有的总线请求都由一个中央仲裁器处理，该仲裁器使用特定的裁决算法来决定哪个设备可以获得总线控制权。
    • 分布式仲裁: 每个设备都有自己的仲裁器，它们通过共享的仲裁总线进行竞争。仲裁通过比较仲裁号的大小来决定谁获得控制权。

集中式仲裁

  1. 链式查询方式:

    • 工作原理: 总线同意信号（BG）沿着一个线性链路从一个设备传递到下一个设备。如果某个设备在BG到达时有总线请求，它会停止BG信号的传递，并获得总线控制权。
    • 优缺点:
      • 优点: 结构简单，易于增删设备，优先级算法简单，容易实现。
      • 缺点: 查询速度较慢，优先级受限于设备的排序，对链路故障敏感。

  2. 计数器定时查询方式:

    • 工作原理: 总线控制部件在接收到总线请求信号后，使用计数器进行查询。通过设备地址线发送地址信号，当计数器的值与设备地址匹配时，该设备获得总线控制权。
    • 优缺点:
      • 优点: 优先级确定灵活，对链路故障不敏感。
      • 缺点: 控制电路复杂，需要更多的控制线。

  3. 独立请求方式:

    • 工作原理: 每个设备都有独立的请求线和同意线，仲裁器根据设备的请求进行排队处理。排队器集中进行优先级判断。
    • 优缺点:
      • 优点: 排队器集中管理，优先级确定灵活。
      • 缺点: 使用的线数多，控制逻辑复杂。

分布式仲裁

  • 工作原理: 每个设备都有自己的仲裁器和唯一的仲裁号。当设备有总线请求时，它将自己的仲裁号发送到共享的仲裁总线上。每个仲裁器比较其仲裁号与总线上的号，若总线上的号大，则撤回请求。最终，获胜的设备的仲裁号保留在总线上。
  • 优点: 不需要中央仲裁器，优先级仲裁策略可以灵活调整。
  • 缺点: 实现较复杂，需要处理多个设备的仲裁号和请求。

总线通信控制（总线的定时）

总线通信控制涉及确保总线上的数据传输过程中的各个步骤能够正确协调。包括以下几个主要阶段：

  1. 申请分配阶段: 主设备向总线发出占用申请，仲裁器决定是否允许其占用总线。
  2. 寻址阶段: 主设备向从设备发送地址和控制命令。
  3. 数据传输阶段: 主设备和从设备进行数据交换。
  4. 总线释放阶段: 主设备完成数据传输后释放总线。

总线通信的方式

  1. 同步通信:

    • 定义: 所有设备使用一个统一的时钟信号来控制数据的传输。数据的发送和接收都依赖于这个公共时钟。
    • 优缺点:
      • 优点: 定时精确，数据传输速率高，时序关系简单，实现容易。
      • 缺点: 对时钟频率敏感，长距离传输中可能出现时钟相移问题，主从设备需要强制同步。

  2. 异步通信:

    • 定义: 数据的传输不依赖于统一的时钟信号，而是通过应答信号或互锁机制来协调传输的时机。总线周期的长度是可变的。
    • 优缺点:
      • 优点: 适合不同速率的设备，传输过程中的时间间隔可以变化，不需要公共时钟。
      • 缺点: 速度相对较慢，需要处理更多的控制信号，可能增加了通信的复杂性。

  3. 半同步通信:

    • 定义: 结合了同步和异步通信的特点，允许一定程度的时钟偏差。数据传输的时间间隔以公共时钟的整数倍为基准，但具体的时序关系可以变化。
    • 优缺点:
      • 优点: 适合不同速率的设备，能够在一定程度上同步时序。
      • 缺点: 实现复杂性较高，对时钟精度要求较低。

  4. 分离式通信:

    • 定义: 各模块在准备数据时不占用总线，充分利用总线的空闲时间。每个模块可以自主申请占用总线进行数据传输。
    • 优缺点:
      • 优点: 充分利用总线，减少等待时间，提高整体系统效率。
      • 缺点: 需要复杂的控制逻辑来管理总线的使用，确保各模块的协调性。
