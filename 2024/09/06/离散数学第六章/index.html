《离散数学及其应用》

图的基础知识

无向图和有向图

无向图 ($ G = (V, E) $) 是由顶点集合 ($ V$ ) 和边集合 ($ E$ ) 组成的图。在无向图中，每条边连接两个顶点，没有方向，两个顶点之间的边是双向的。换句话说，对于无向图中的每一条边 ($\{u, v\}$)，顶点 ($u$) 和 ($v$) 是相邻的。

有向图 ($ G = (V, E)$ ) 由顶点集 ( $V $) 和有向边（或弧）集 ($ E$ ) 组成。在有向图中，每条边有方向，即每条边是从一个顶点指向另一个顶点的。在有向图中，我们使用有序对 ($(u, v)$) 来表示边，其中 ($ u$ ) 是起点，($v $) 是终点。一个有向图中的环是起点和终点相同的有向边。

图的术语

邻接 / 相邻

  • 单个顶点的邻居: 如果顶点 ($ u$ ) 和顶点 ( $v $) 通过一条边连接，则称 ( $u $) 和 ( $v$ ) 是相邻的。顶点 ( $u$ ) 的邻居集合记为 ( $N(u)$ )，即所有与 ( $u$ ) 相连的顶点的集合。

$
N(u) = \{ v \in V \mid \text{存在一条边} \ \{u, v\} \text{或有向边} \ (u, v) \text{连接} \}
$

  • 顶点集合的邻居: 对于顶点集合 ( A )，其邻居集合为所有属于 ( A ) 中顶点的邻居的集合：

$
N(A) = \bigcup_{u \in A} N(u)
$

度

  • 无向图中的度: 顶点 ( u ) 的度 ( $\text{deg}(u) $) 是与 ( u ) 相连接的边的数量。

  • 有向图中的度:

    • 入度 ($ \text{deg}^{-}(u)$ ) 是以 ($ u $) 为终点的边的数量。
    • 出度 ($ \text{deg}^{+}(u)$ ) 是以 ($ u $) 为起点的边的数量。

公式：
$
\sum_{u \in V} \text{deg}^{-}(u) = \sum_{v \in V} \text{deg}^{+}(v) = |E|
$

这里的 ($ |E| $) 表示边的总数。由于每条边都有一个起点和一个终点，因此入度总和等于出度总和。

握手定理

在无向图 ( $G = (V, E) $) 中，边数 ($ m$ ) 的总度数为：

$
\sum_{u \in V} \text{deg}(u) = 2m
$

这是因为每条边都被计算两次（一次在每个端点的度数中），因此度数总和是边数的两倍。

特殊图

完全图

完全图 ($ K_n$ ) 是一个每对不同顶点都有边相连的图，其中 ($ n$ ) 是顶点的数量。换句话说，完全图中的每个顶点都与图中的所有其他顶点相连。

$
K_n
$

圈图

圈图 ($ C_n$ ) 是一个图，其中每个顶点仅与其相邻的两个顶点连接，形成一个环。圈图中的顶点数 ($ n$ ) 必须大于或等于3。

$
C_n
$

轮图

轮图 ( $W_n $) 是在圈图 ( $C_n $) 的基础上增加一个额外的顶点，与圈图中的所有顶点相连。轮图中的顶点数 ($ n$ ) 必须大于或等于3。

$
W_n
$

立方体图

立方体图 是一个立方体的图形表示，其中每个顶点有一个二进制编码，相邻顶点的二进制编码仅有一个位不同。例如，对于4个顶点的立方体图，相邻的顶点对编码仅在一个位置上有不同的位。

二分图

二分图 是将顶点集分为两个子集 ( $U $) 和 ($ V$ )，使得每条边都连接这两个子集中的顶点。即图中的顶点仅与另一个子集中的顶点相连，而不与同一子集中的顶点相连。

完全二分图

完全二分图 是一种特殊的二分图，其中子集 ( U ) 中的每个顶点都与子集 ( V ) 中的每个顶点相连。

子图

子图 是从原图中通过删除一些顶点和对应边所得到的图。子图是原图的一个子集，包括原图中某些顶点和边的集合。换句话说，子图是原图的一部分，不包含原图中所有的顶点和边。

连通性：

适合无向图。

通路

在图论中，通路（path）是指图中从一个顶点到另一个顶点的路径。如果图中存在从顶点 (a) 到顶点 (c) 的一条路径，这条路径称为通路。通路的长度由路径上的边的数量决定。以下是几个通路的示例：

  • 从顶点 (a) 到 (c) 的通路可以是 ((a, b, c)) 或 ((a, d, c))。
  • ((a, c)) 不是通路，因为在这个例子中没有直接的边连接 (a) 和 (c)。

一个回路（cycle）是指通路的起点和终点是同一个顶点，并且通路的长度不为0。

连通性

图的连通性指的是图中任意两个顶点之间都存在通路。如果图中的每个顶点都可以通过某种路径互相连接，那么这个图被称为连通图（connected graph）。换句话说，图中没有孤立的点，任意两点之间都可以通过某些路径相连。

割点和割边

在一个连通图中，如果去掉某个顶点后，图被分割成两个或多个连通部分，那么这个顶点叫做割点（cut vertex）或关节点（articulation point）。类似地，如果去掉某条边后，图被分割成两个连通部分，那么这条边叫做割边（cut edge）或桥（bridge）。

点割集（点分割集）

点割集（vertex cut set）是指所有割点的集合。即，点割集是使图失去连通性的所有顶点的集合。

连通度

连通度（connectivity）是一个图论概念，用于描述一个图的“坚固”程度，即需要删除的最少顶点数目，使得图变得不连通。具体定义为，图的连通度是使图变得不连通所需删除的最少顶点数。

  • 连通度记作 ($\kappa(G)$)，其中 (G) 是图。
  • 对于一个完全图 ($K_n$)，其连通度 ($\kappa(K_n) = n - 1$)。这是因为，要使完全图 ($K_n$) 失去连通性，我们必须删除除了一个顶点外的所有顶点。
  • 对于不连通的图，($\kappa(G) = 0$)，因为图本身已经不是连通的。

边隔集和边连通度

边隔集（edge cut set）是使图失去连通性的所有边的集合。

边连通度（edge connectivity）记作 ($\lambda(G)$)，是使图失去连通性所需删除的最少边数。

  • 对于一个完全图 ($K_n$)，其边连通度 ($\lambda(K_n) = n - 1$)。这是因为，要使完全图 ($K_n$) 失去连通性，我们必须删除与某个顶点连接的所有边。

  • 通路：图中从一个顶点到另一个顶点的路径。
  • 连通性：图中每对顶点之间都有通路的性质。
  • 割点和割边：删除某个顶点或边使图分裂成多个连通图的点或边。
  • 点割集：使图失去连通性的所有割点的集合。
  • 连通度：使图变得不连通所需删除的最少顶点数。
  • 边隔集：使图失去连通性的所有边的集合。
  • 边连通度：使图变得不连通所需删除的最少边数。

1. 连通度与边连通度的关系

对于任何图 ( G )：

$ \kappa(G) \leq \lambda(G) $

这表明图的连通度（顶点连通度）总是小于或等于边连通度（边的连通度）。这可以通过以下几种情况来理解：

  • 不连通图：对于不连通图，无论是连通度还是边连通度，都是0。即：($\kappa(G) = \lambda(G) = 0$)。
  • 有割点集的图：如果图中存在割点集（即至少存在一个割点），则 ($\kappa(G) = \lambda(G) = 1$)。
  • 没有割点集的连通图：在这种情况下，我们考虑删除的边 ($(x, y)$) 对于连通性产生的影响。删除一条边可能导致图的连通性下降，但不一定需要删除所有与边 ($(x, y)$) 相连的顶点。删除边的数量一定少于或等于删除点的数量，从而有 ($\kappa(G) \leq \lambda(G)$)。

2. 连通度与顶点度数的关系

对于一个图 (G = (V, E))，其中 (G) 不是完全图且 ($|V| \geq 3$)：

$ \kappa(G) \leq \min_{v \in V} \deg(v) $
$ \lambda(G) \leq \min_{v \in V} \deg(v) $

这里的 ($\deg(v)$) 是顶点 ($v$) 的度数，即与 ($v$) 相连的边的数量。这个结论的意思是，连通度和边连通度都不能超过图中最小的顶点度数。原因是，如果图中的任何顶点度数最小为 ($k$)，则删除这个顶点的所有边只能分割图成至多 ($k$) 个部分。

3. 有向图的连通性

在有向图中，连通性被分为强连通和弱连通：

  • 强连通：如果在有向图中，从任何一个顶点可以通过有向边到达任何其他顶点，则该图称为强连通。例如，图1展示了强连通的情况。

  • 弱连通：如果将图中的所有有向边都视为无向边，图仍然连通，则该图称为弱连通。例如，图2展示了弱连通的情况。

强连通分支（strongly connected component）是指一个有向图的子图，如果它是强连通的，则称为强连通分支或强分支。

4. 图同构

同构图（isomorphic graphs）是指两幅图在顶点的排列或重标号上是等价的，即它们具有相同的图结构。简单回路的长度是图同构的一个不变量，意味着两个同构的图在所有相同长度的简单回路上是相同的。

  • 对于一个图 (G) 的连通度和边连通度，通常小于或等于图中所有顶点度数的最小值。

最短通路问题：BFS 和 DFS

最短通路问题主要关注如何在图中找到从起点到终点的最短路径。这里介绍了两种经典的算法：广度优先搜索（BFS）和深度优先搜索（DFS）。这两种算法在解决最短路径问题时的应用有所不同。

广度优先搜索（BFS）

BFS 是用于寻找无权图（即图中所有边的权重均相等）中最短路径的一种算法。它的基本思想是从起始节点开始，逐层探索图的节点，直到找到目标节点。它保证了找到的路径是最短的。

BFS 算法步骤：

  1. 初始化：创建一个队列，初始时将起点加入队列。同时，记录已经访问过的节点，避免重复访问。
  2. 循环处理：
    • 从队列中取出当前节点。
    • 检查当前节点是否为目标节点。如果是，路径找到。
    • 将当前节点的所有未访问邻居加入队列，并标记为已访问。
  3. 结束条件：队列为空或找到目标节点时停止。

深度优先搜索（DFS）

DFS 是一种用于寻找有权图中最短路径的算法。它通过递归或栈结构，沿着图的每条路径进行深度优先的探索。

在图中，如果边有不同的权重，DFS 算法需要维护一个记录最短路径的表格，其中包括当前顶点到起点的最短距离和前一个节点。DFS 的实现通常包括以下步骤：

  1. 初始化：
    • 创建一个表格记录每个节点的最短距离和前一个节点。
    • 从起始节点开始，初始化其距离为0，其余节点距离为无穷大。
  2. 递归处理：
    • 从当前节点开始，更新邻接节点的距离。
    • 选择距离最小的未处理节点进行递归。
  3. 结束条件：当所有节点都被处理完毕，或者找到目标节点时停止。

平面图

平面图的定义

一个平面图指的是一个图可以在平面上画出，并且其边没有在顶点以外的地方相交的图。也就是说，图的所有边在其平面表示中都是分开的，除了在公共顶点处交汇。一个图的平面表示就是该图的一种绘制方式，其中边不交叉。

欧拉公式

对于连通的平面简单图 ( $G$ )，有一个重要的定理叫做欧拉公式，它描述了该图的顶点数、边数和面数之间的关系。设 ( $u$ ) 表示顶点数，( $e$ ) 表示边数，( $r$ ) 表示图的面数，则欧拉公式如下：
$
r = e - u + 2
$
其中，面数 ( r ) 是指图在平面表示中形成的区域数，包括外部的无限区域。

示例

考虑一个具体例子：假设一个图 ($ G$ ) 有 8 个顶点、12 条边。在该图的平面表示中，观察到图有 6 个面（包括外部的无限面），应用欧拉公式：

$
r = e - u + 2 = 12 - 8 + 2 = 6
$

推论1：平面图的边界

对于连通平面简单图，如果图中顶点数 ( u \geq 3 )，则该图的边数 ( e ) 满足以下不等式：
$
e \leq 3u - 6
$

推论1的证明

  1. 根据握手定理，图中的边数与顶点的度数之间存在如下关系：
    $
    2e = \sum_{v \in V} \deg(v)
    $
    其中 ($ \sum_{v \in V} \deg(v) $) 表示所有顶点的度数之和。

  2. 在连通平面简单图中，每个面至少由 3 条边组成，因此有：
    $
    \sum_{v \in V} \deg(v) \geq 3r
    $
    也就是说，所有顶点的度数之和至少为 3 倍的面数。

  3. 根据欧拉公式 ($ r = e - u + 2$ )，将 ( $r$ ) 替换为边数和顶点数的表达式：
    $
    2e \geq 3(e - u + 2)
    $

  4. 简化上式，可以得到：
    $
    e \leq 3u - 6
    $
    这就是推论 1 的证明。

推论2：平面图中的低度顶点

对于一个连通平面简单图，图中至少有一个顶点的度数不超过 5。

推论2的证明

假设图中所有顶点的度数都大于 5，即每个顶点的度数至少为 6，根据握手定理：
$
2e = \sum_{v \in V} \deg(v) \geq 6u
$
因此有 ($ e \geq 3u$ )，但这与推论 1 的 ( $e \leq 3u - 6 $) 相矛盾。因此，假设不成立，说明图中至少有一个顶点的度数不超过 5。

推论3：平面图中的度数限制

若一个连通平面简单图 ( G ) 中的每个顶点的度数至少为 4，且顶点数 ( $u \geq 3 $)，则该图的边数 ( $e $) 满足：
$
e \leq 2u - 4
$

推论3的证明

  1. 由握手定理可知：
    $
    2e = \sum_{v \in V} \deg(v) \geq 4u
    $
    因为每个顶点的度数至少为 4。

  2. 因此有 ($ e \geq 2u$ )。

  3. 使用欧拉公式 ( $r = e - u + 2$ ) 替换面数 ($ r$ )，并应用类似推导，可以得到：
    $
    e \leq 2u - 4
    $

判断一个图是否是平面图

通过欧拉公式及其推论，我们可以判断一个图是否是平面图。例如，对于给定的一个图，如果它的边数和顶点数满足推论 1 或推论 3 的不等式，则该图可能是平面图。如果不满足这些条件，则该图不是平面图。

一个典型的例子是完全二分图 ( $K_{3,3} $)。此图有 6 个顶点和 9 条边，无法通过平面表示不交叉绘制。因此，可以通过使用推论 3 来判断此图不是平面图。虽然通过推论 1 并不能排除 ( $K_{3,3} $) 作为平面图的可能性，但推论 3 确定了其无法满足边数和顶点数的关系。

康拉图斯基定理

康拉图斯基定理指出，一个图如果包含完全图 ( $K_5$ ) 或完全二分图 ($ K_{3,3}$ ) 的子图，则该图不是平面图。

  • ($ K_5$ )：5 个顶点的完全图，每一对顶点都由一条边相连，总共 10 条边。
  • ($ K_{3,3}$ )：3 个顶点和 3 个顶点的完全二分图，两个独立集合中的每一个顶点都与另一个集合中的所有顶点相连，总共有 9 条边。

因此，如果一个图包含 ( $K_5$ ) 或 ( $K_{3,3} $) 作为子图，那么该图无法在平面上绘制而边不交叉，也就是说它不是平面图。

  1. 平面图是可以在平面上表示的图，其中的边不交叉。
  2. 欧拉公式给出了平面图的顶点数、边数、和面数之间的关系。
  3. 推论 1 和 3 提供了判断图是否是平面图的边数与顶点数的限制。
  4. 康拉图斯基定理提供了一种通过图的子图结构（例如 ($ K_5 $) 和 ($ K_{3,3}$ )）来判断图是否是平面图的方法。

图着色

图着色是一种给定简单图 ( G ) 的每个顶点分配颜色的方式，使得任何两个相邻的顶点都具有不同的颜色。这一过程确保图的结构不违反相邻顶点颜色不同的约束。

图的着色数

图的着色数 ( \chi(G) ) 是给图着色所需的最少颜色数。它反映了图结构的复杂性，也就是说，越复杂的图需要越多的颜色来满足不相邻顶点颜色不同的要求。

例子：

假设我们有一个简单的三角形图 ( $C_3$ )，其中每个顶点都与其他两个顶点相连。在这种情况下，需要三种颜色来确保所有相邻顶点的颜色都不同，因此该图的着色数 ($ \chi(C_3) = 3 $)。

四色定理

四色定理是图论中的一个经典结论，内容为：

任意平面图的着色数不超过4，即任何平面图都可以用至多四种颜色对其进行着色，使得相邻顶点的颜色不同。

这个定理最早由 Francis Guthrie 在1852年提出，并于1976年通过计算机辅助证明。它说明了在任何平面图中，无论图的形状多么复杂，都能通过四种或更少的颜色完成顶点着色。

四色定理的思考过程：

  1. 平面图的构造：平面图是可以在平面上绘制的图，图中的边互不交叉。
  2. 增添顶点：为了简化平面图的着色，可以在每个面的内部添加一个顶点，使其与该面边界上的所有顶点相连，这样将原来的图划分成若干个三角形面。每个面只需要考虑三个顶点相邻的情况，保证每个三角形内的顶点着色不同。

证明思路（非正式）：

  • 将任意平面图分解成多个三角形，这些三角形可以通过添加顶点构造。
  • 对于每一个面，无论该面是奇数边还是偶数边，通过递归将其着色，直到所有的顶点都完成着色。
  • 最终，只需要3到4种颜色就可以确保每个顶点的着色满足条件。

图形构造示例：

  1. 将每个面的中心添加一个顶点，并连接到该面边界的所有顶点，使得每个面都成为一个三角形。
  2. 在每个三角形中，使用3种颜色着色内部顶点，然后逐步处理相邻的三角形。对于偶数边的图，可能需要引入第4种颜色。

为什么最多只需4种颜色？

对于平面图，基于三角形的构造方式，每个顶点最多与4个其他顶点相连（通过相邻三角形的构造）。因此，最多需要4种颜色来着色相邻的顶点。这个结论由四色定理得出，并被广泛用于地图、网络拓扑等问题的解决。
