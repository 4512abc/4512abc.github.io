《离散数学及其应用》

树的概述

树的定义

树是图论中的一种特殊类型的无向图。具体来说，树满足以下两个条件：

  1. 无回路：树中不存在任何简单回路。
  2. 连通性：树是连通图，即在树中的任意两顶点之间至少存在一条路径。

根据这些性质，一个图是树当且仅当以下两个条件同时满足：

  • 任意两个顶点之间存在唯一的一条简单路径。
  • 图中没有回路。

树通常用符号 ( T ) 表示。

森林

如果一个图由多个互不连通的树组成，则该图被称为森林。森林是多个树的集合，且其中的每个连通分量都是一棵树。

有根树

树的基本定义是无向的且无根的。然而，通过指定一个特定的顶点为根，可以将一棵无向树转化为有根树。在有根树中，每个顶点有了明确的层次关系，边的方向从根出发指向其他顶点。

有根树的定义

有根树是指定一个顶点作为根的树，并且将树的每条边的方向规定为从根指向其他顶点，形成了一种具有方向性的结构。这样一棵树可以视作一个有向图，其中所有路径从根节点出发。

有根树中的基本概念

在有根树中，由于根的存在，顶点之间有了明显的层次和关系，可以定义一系列相关概念：

  1. 父母节点（Parent Node）：

    • 若顶点 ( u ) 通过一条从顶点 ( v ) 出发的边连接到 ( v )，则 ( v ) 称为 ( u ) 的父母节点。即在根的方向上，紧邻上方的节点是当前节点的父母。

    示例：若顶点 ( c ) 通过边与 ( b ) 相连，则 ( b ) 是 ( c ) 的父母节点。

  2. 孩子节点（Child Node）：

    • 若顶点 ( u ) 连接至顶点 ( v )，则 ( v ) 称为 ( u ) 的孩子节点。换句话说，在根的方向上，紧邻下方的节点是当前节点的孩子。

    示例：若顶点 ( g ) 通过边与 ( h, i, j ) 相连，则 ( h, i, j ) 是 ( g ) 的孩子节点。

  3. 兄弟节点（Sibling Node）：

    • 若两个节点有相同的父母节点，则它们互为兄弟节点。

    示例：若顶点 ( h ) 和 ( i ) 都与 ( g ) 相连，则 ( h ) 和 ( i ) 是兄弟节点。

  4. 祖先节点（Ancestor Node）：

    • 若从根节点开始有一条路径可以经过若干节点到达当前节点，则这些路径上的所有节点称为该节点的祖先。

    示例：顶点 ( e ) 的祖先为顶点 ( c, b, a )。

  5. 后代节点（Descendant Node）：

    • 若从当前节点可以通过路径到达某个节点，则该节点是当前节点的后代。后代与祖先相对。

    示例：顶点 ( b ) 的后代为顶点 ( c, d, e )。

  6. 叶节点（Leaf Node）：

    • 没有任何孩子的节点称为叶节点。叶节点是树的终端节点。

    示例：顶点 ( d, e, f, k, i, l, m ) 都是叶节点。

  7. 内点（Internal Node）：

    • 有孩子的节点称为内点。内点是除叶节点外的所有节点。

    示例：顶点 ( a, b, g, c, h, j ) 都是内点。

  8. 子树（Subtree）：

    • 以某个节点为根，包含该节点及其所有后代的部分树称为子树。

    示例：以顶点 ( g ) 为根的子树包含节点 ( g, h, i, j )。

m叉树与二叉树

m叉树

在有根树中，若每个内点的孩子数都小于 ( m )，则称该树为m叉树。如果每个内点恰好有 ( m ) 个孩子，则称该树为满m叉树。

  • m叉树：内点的孩子数最多为 ( m )。
  • 满m叉树：每个内点恰有 ( m ) 个孩子。

当 ( m = 2 ) 时，m叉树即为二叉树。二叉树是计算机科学中的重要数据结构，其每个节点最多有两个孩子，通常称为左孩子和右孩子。

有序根树

有序根树是一种特殊的有根树，其特点是对每个节点的孩子有明确的顺序。通常，规定每个节点的孩子从左到右排列。

二叉树中的左右子树

在二叉树中，有序根树的概念尤为重要。二叉树的每个节点有至多两个孩子：

  1. 左子树：以左孩子为根的子树称为左子树。
  2. 右子树：以右孩子为根的子树称为右子树。

这种左右子树的结构对于许多二叉树的应用，如二叉搜索树和堆结构，具有重要意义。

树的性质

1. 树的边数

在图论中，树的一个重要性质是：具有 (n) 个顶点的树必然含有 (n-1) 条边。

这个性质可以从树的定义中推导出来：

  • 树是连通图且没有回路。
  • 对于 (n) 个顶点的连通无向图，如果再加入一条边，会形成回路。因此，最少的边数就是 (n-1) 条。

可以通过数学归纳法证明这一性质：

证明：

  1. 基步：当 (n=1) 时，树中只有一个顶点，没有边，满足公式 (n-1 = 0)。
  2. 归纳假设：假设对于任意具有 (k) 个顶点的树，其边数为 (k-1)。
  3. 归纳步骤：对于一个具有 (k+1) 个顶点的树，如果去掉一个叶子节点，那么剩下的子树就是一个具有 (k) 个顶点的树，根据归纳假设，剩下的子树有 (k-1) 条边。再将被去掉的叶子和它的唯一一条边重新加上，整个树就有 (k) 条边。由此可得：具有 (k+1) 个顶点的树含有 (k+1-1 = k) 条边。

因此，对于任意 (n) 个顶点的树，它的边数是 (n-1)。

2. 满m叉树的性质

对于满 (m) 叉树，即每个内点都有 (m) 个孩子的树，存在以下重要性质：

  • 顶点数：满 (m) 叉树若有 (i) 个内点，则总顶点数 (n = mi + 1)。

推导过程：

  1. 每个内点有 (m) 个孩子，因此 (i) 个内点总共有 (mi) 个孩子（即边数为 (mi) 条）。
  2. 根据树的边数公式 (n-1 = mi)，我们得出 (n = mi + 1)。

因此，带有 (i) 个内点的满 (m) 叉树有 (n = mi + 1) 个顶点。

3. 满m叉树中的内点、顶点、树叶的关系

在满 (m) 叉树中，内点、顶点、树叶之间存在以下关系：

$
\begin{aligned}
\text{若有 } n \text{ 个顶点，则：} \quad & i = \frac{n-1}{m}, \quad l = \frac{(m-1) \cdot n + 1}{m} \\
\text{若有 } i \text{ 个内点，则：} \quad & n = mi + 1, \quad l = (m-1) i + 1 \\
\text{若有 } l \text{ 个树叶，则：} \quad & n = \frac{ml - 1}{m-1}, \quad i = \frac{l - 1}{m-1}
\end{aligned}
$

公式推导：

  1. 已知内点数 (i)：

    • 根据满 (m) 叉树的性质，顶点数 (n = mi + 1)。
    • 树叶数 ($l = (m-1)i + 1$)（因为总顶点数为 (n)，其中 (i) 是内点，而 (l) 是叶节点）。

  2. 已知顶点数 (n)：

    • 内点数 ($i = \frac{n-1}{m}$)。
    • 树叶数 ($l = \frac{(m-1) \cdot n + 1}{m}$)。

  3. 已知树叶数 (l)：

    • 内点数 ($i = \frac{l-1}{m-1}$)。
    • 顶点数 ($n = \frac{ml-1}{m-1}$)。

推导解释：

  • 当已知树叶数 (l) 时，通过以下关系可以推导出内点数：
    $
    n = l + i \quad \text{且} \quad n = mi + 1
    $
    联立这两个公式，可得：
    $
    i = \frac{l - 1}{m-1}
    $
    再结合 (n = l + i) 得到：
    $
    n = l + \frac{l-1}{m-1} = \frac{ml-1}{m-1}
    $

4. 平衡的m叉树

平衡的 (m) 叉树的定义涉及到树的结构是否均匀。平衡树中的一个重要特性是：

平衡树中，每个内点的子树大致包含相同的路径长度。

在讨论平衡树时，常常用到层与高度的概念。

  • 层：在有根树中，顶点 (u) 的层是指从根节点到该顶点的唯一简单路径的长度（边数）。
  • 高度：有根树的高度是指树中所有顶点的层数的最大值。即从根到树叶的最长路径的长度称为树的高度。

5. m叉树的高度与树叶数

在高度为 (h) 的 (m) 叉树中，树叶的数量满足以下不等式：

$
l \leq m^h
$

即，树的高度越高，树叶的数量最多可达到 (m^h) 个。该不等式是基于满 (m) 叉树的性质得出的。

此外，树叶数与高度之间存在以下关系：

$
h \geq \left\lceil \log_m l \right\rceil
$

这意味着，如果已知树叶的数量 (l)，则树的高度 (h) 至少为 ($ \log_m l$ ) 的上取整值。这是因为在满 (m) 叉树中，最大可能的树叶数量为 ($m^h$)，因此高度至少为 ($\log_m l$)。

树的应用

前缀码

前缀码是一种非固定长度的编码方式。设想我们有26个字母，每个字母可以用固定长度的5个比特（bit）来表示，例如：

  • $00000：a$
  • $00001：t$

当我们传输单词at时，就可以传递比特串00000001，这种方式虽然简单，但固定长度会造成一些浪费。为了解决这个问题，我们可以使用前缀码，一种允许不同字符具有不同长度编码的方式。

例如，假设我们为字符分配以下编码：

  • e的编码为：0
  • a的编码为：10
  • t的编码为：110
  • s的编码为：111

当我们传输单词teas时，对应的比特串为：

t e a s
110 0 10 111

即传递110010111。

前缀码的优势在于它是一种无前缀的编码方案，即没有任何字符的编码是其他字符编码的前缀。这使得接收方能够逐步解码输入，直到遇到终止符，即树的叶节点。

哈夫曼编码

哈夫曼编码是一种前缀码的经典实现。它基于字符出现的频率为字符分配变长编码，频率越高的字符分配的编码越短，从而压缩信息量。

首先，统计给定字符串中每个字符出现的频率。例如，对于句子：

2. 构建哈夫曼树

接下来，根据频率构建哈夫曼树。构建的基本思想是：频率最低的两个字符组合成一个新的节点，该节点的频率等于两个字符频率之和，然后重复这个过程，直到生成整棵树。

  1. 选择频率最小的两个字符，如T (0.03)和, (0.03)，它们的父节点频率为0.06。

  2. 将这个新节点插入到频率列表中，并继续选择最小的两个节点。

  3. 重复上述过程，直到所有节点都合并成一棵树。

通过这种方式，我们最终得到一棵哈夫曼树。

树的遍历

在树结构中，最常用的遍历方法有前序遍历、中序遍历和后序遍历，这些方法用于访问树中的节点。二叉树的每个节点都有最多两个子节点，分别为左子节点和右子节点。下面我们来看每种遍历方式的规则：

  • 先访问当前节点，再访问左子节点，最后访问右子节点。

3. 不同记法的应用场景

  • 中缀记法是最常用的表示法，符合我们日常的书写和阅读习惯，但在计算机中处理时需要考虑运算符优先级和括号的匹配。

  • 前缀记法（波兰记法）不需要括号来处理运算符优先级，但需要从右向左进行计算，这在手工计算时可能不直观。

  • 后缀记法（逆波兰记法）也不需要括号，并且非常适合用栈结构进行表达式求值。在计算器中，后缀记法经常被用来处理复杂的算术表达式。

生成树

定义

生成树（Spanning Tree）是图论中的一个重要概念。给定一个连通无向图 ( G )，其生成树是一个子图 ( T )，满足以下两个条件：

  1. 包含图 ( G ) 的所有顶点。
  2. 没有环路，并且生成树是一个连通图。

换句话说，生成树保留了原图的所有顶点，但删去了一些边，使得图变成了一棵树。如果一个图是连通的，它必定存在至少一棵生成树。

生成树的性质：

  • 对于一个有 ( n ) 个顶点的连通图，其生成树恰好有 ( n - 1 ) 条边。
  • 一个图可能有多个生成树，但每个生成树都具有相同的顶点数和相同数量的边。

  1. a-b, b-d, d-c
  2. a-c, c-d, d-b

两者都包含所有顶点，并且没有环路。

深度优先搜索（DFS）构建生成树

深度优先搜索是一种通过递归或栈结构在图中遍历节点的方法，常用于构建生成树。其基本思想是沿着一条路径尽可能深入地遍历，直到没有可访问的节点为止，再回溯并访问其他分支。

算法步骤

  1. 选择一个起始顶点作为根节点。
  2. 从当前顶点出发，沿着一条边遍历到下一个未访问过的顶点。
  3. 标记该顶点为已访问，并继续沿着未访问的边向下递归，直到无路可走。
  4. 回溯到上一个顶点，并探索其他未访问的路径。
  5. 重复上述步骤，直到遍历完整个图。

广度优先搜索（BFS）构建生成树

广度优先搜索是一种通过队列逐层访问节点的方法，通常用于寻找最短路径，也可以用于构建生成树。

算法步骤

  1. 选择一个起始顶点作为根节点，加入队列。
  2. 从队列中取出一个顶点，访问其所有未访问的相邻顶点。
  3. 将这些相邻顶点标记为已访问，并依次加入队列。
  4. 重复上述步骤，直到队列为空。

生成树与其他概念的类比

生成树的概念可以与哈赛图、传递闭包等关系类比：

  • 哈赛图：哈赛图是有向无环图的一种特殊形式，用于描述偏序关系。生成树通过遍历图的节点，确保了无环路，这与哈赛图的无环结构类似。

  • 传递闭包：在关系论中，传递闭包是将传递性质扩展到所有关系对，保证连通性。生成树通过连通图的所有顶点，实现了一种隐含的传递性。

通过这些类比，生成树在图论中的连通性和无环特性与其他领域的概念息息相关。

最小生成树

定义

在图论中，最小生成树（Minimum Spanning Tree, MST） 是一个特殊的生成树，其边的权重之和最小。也就是说，对于一个连通加权图，最小生成树是包含图中所有顶点的生成树，其中边的权重和最小。

最小生成树的重要性

最小生成树在网络设计、优化问题和图形处理等多个领域中非常有用。例如，在计算机网络中，最小生成树可以用于设计最短的网络连通路径以最小化成本。

常用算法

最小生成树的两个经典算法是 普林算法（Prim’s Algorithm） 和 克鲁斯卡尔算法（Kruskal’s Algorithm）。这两个算法都可以用来求解最小生成树，但采用的策略和方法不同。

1. 普林算法（Prim’s Algorithm）

普林算法是一种贪心算法，从一个顶点开始，逐步构建最小生成树。算法步骤如下：

  1. 初始化：选择一个起始顶点，将其标记为已访问，并初始化一个空的生成树。
  2. 选择最小边：从已访问的顶点集合中，选择权重最小的边，将此边加入生成树。
  3. 更新：将新加入生成树的顶点标记为已访问，并更新候选边（即连接已访问顶点和未访问顶点的边）。
  4. 重复：重复选择最小边和更新过程，直到所有顶点都被访问。

2. 克鲁斯卡尔算法（Kruskal’s Algorithm）

克鲁斯卡尔算法也是一种贪心算法，通过逐步添加边来构建最小生成树，算法步骤如下：

  1. 初始化：将所有边按照权重升序排列，并初始化一个空的生成树。
  2. 选择最小边：从排好序的边集合中选择权重最小的边，检查这条边是否会形成回路。如果不会形成回路，则将这条边加入生成树。
  3. 重复：重复选择边的过程，直到生成树包含 ( n-1 ) 条边（( n ) 是顶点数）。

普林算法与克鲁斯卡尔算法的比较

  • 普林算法：适用于边数多的图，尤其适用于密集图。它通过逐步扩展的方式构建生成树，从一个顶点出发，逐步加入最小边。

  • 克鲁斯卡尔算法：适用于边数少的图，尤其适用于稀疏图。它通过边的排序和查找-并集数据结构来检测回路，逐步添加最小边。

复杂度分析

  • 普林算法：

    • 时间复杂度：($ O(E \log V)$ )（使用优先队列实现），其中 ( E ) 是边数，( V ) 是顶点数。

  • 克鲁斯卡尔算法：

    • 时间复杂度：( $O(E \log E) $)（由于边的排序），其中 ( E ) 是边数。
